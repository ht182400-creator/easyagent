## release-publish.bat Token 自动加载修复

### 问题
- option 2 选择后，即使 token 文件存在，打印了"Token auto-loaded"后仍然弹出输入提示
- 根因：`set /p GITHUB_TOKEN=<scripts\.release_token` 中的 `<` 重定向写在 `if (...) else (...)` 括号块内，cmd.exe 解析器误把 `) else (` 的 `)` 视为重定向目标文件路径的一部分
- 这是 cmd 批处理的经典陷阱：if 括号块内使用 `<` / `>` 重定向会导致括号匹配混乱

### 修复
- 将 `set /p ... <file` 改为 `for /f "usebackq delims=" %%t in ("file") do set "VAR=%%t"` 方式读取
- 逻辑重构：先尝试读文件 → 成功则 `goto :TOKEN_READY` 跳过提示 → 否则 fall through 到手动输入
- `:TOKEN_READY` 标签后统一检查 GITHUB_TOKEN 是否为空

## release-publish.bat Release 已存在时自动上传

### 问题
- Release v0.4.0 已存在时 curl POST 返回 422 "already_exists"，脚本直接跳到手动上传
- 手动上传指引未提及 GitHub 网页 25MB 文件大小限制，EXE ~85MB 根本无法通过浏览器上传

### 修复
1. **422 自动处理**：检测到 "already_exists" 后，通过 GET `/releases/tags/:tag` 获取已有 Release 的 upload_url，然后自动上传文件
2. **提取 upload_url 统一到 `:EXTRACT_UPLOAD_URL` 标签**：新建和已有 Release 两条路径共享
3. **手动上传指引更新**：明确警告 25MB 限制，提供 gh CLI 命令作为替代方案
