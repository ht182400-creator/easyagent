## CI Sync Pipeline Data rebase 冲突根治 (10:32)
- 问题：发版时两个 commit 几乎同时推送 → 两个 CI sync job 并行 unified-sync → pull --rebase merge conflict
- v0.6.14 CI 验证：Release 全绿（Build Desktop/Build Web 修复生效），CI sync-pipeline-data 因冲突失败
- 根治方案：ci.yml Commit pipeline data 步骤改为冲突时 rebase --abort → reset --hard origin/main → 重跑 unified-sync → commit → push
- 核心原理：管线文件是全量生成的，reset 后重跑 unified-sync 得到基于最新远程 + 本轮测试报告的完整数据，无数据丢失

## v0.6.20 release-publish.bat rebase 冲突修复 + 根因根治 (18:43-18:48)
- release-publish.bat Step 3 中 `git pull --rebase origin main` 因 4 个 pipeline 文件冲突失败
- **真根因**：release.mjs 使用 `git add .`，将 post-commit hook 遗留的脏管线文件纳入 commit
  - post-commit hook → update-progress.mjs → unified-sync.mjs --force 修改管线 JSON
  - 这些脏文件留在工作区，下次 `git add .` 时被暂存
  - release commit 包含管线数据 ≠ origin/main（CI auto-sync 推送的）→ rebase 冲突
- **根治**：`git add .` 前先 `git checkout HEAD -- docs/pipeline/` 恢复管线文件到 HEAD 状态
  - commit 后再次 `git checkout HEAD -- docs/pipeline/` 清理 post-commit hook 新产生的修改
  - 文件：scripts/release.mjs 第 359-381 行
- 冲突解决：`git checkout --ours` → `git rebase --continue` → `git tag -f v0.6.20 HEAD`
- push main + tag 成功 ✅，Release workflow 自动触发（push 事件）
- Steps 3.5-6 未执行，需重新运行 `release-publish.bat` 选 `[0] skip` 继续
