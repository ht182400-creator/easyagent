## 发版 push 竞态条件根治 (00:25-00:28)
- 问题：release-publish.bat 每次 git push origin main 都被 rejected (non-fast-forward)
- 根因：发版构建期间（3-5分钟），CI pipeline sync 推了管线数据 commit 到 main
- 第二层问题：git hooks(post-commit)自动运行 unified-sync 修改 docs/pipeline/*.json，导致 pull --rebase 失败
- 修复：scripts/release.mjs 和 release-publish.bat 两处的 push 前加 `git stash → git pull --rebase origin main → git stash pop`
- 安全：远程 CI 只改 pipeline JSON，本地只改 version/CHANGELOG，文件集不相交，永无冲突
- 文档：docs/47_发版推送竞态条件分析与根治.md
