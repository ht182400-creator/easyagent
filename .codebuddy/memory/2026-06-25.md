### 遇到的问题及解决
1. **Node 24 版本拦截**: preinstall.cjs 拦截 Node >= 24。在 `build.bat` 开头添加 `set EASYAGENT_SKIP_NODE_CHECK=1` 绕过。
2. **cmd 路径问题**: "Claude Code  CN" 目录在 cmd 中无法 cd。通过创建项目内的 bat 脚本并用 `cd /d "%~dp0"` 解决。
3. **release.mjs 交互确认**: readline 无法通过管道输入。用文件重定向 (`< _answers.txt`) 解决第一个确认，第二个确认失败后手动执行 git add/commit/tag/push。
4. **build.bat goto 标签**: 重写文件时误将 `:LABEL` 写成 `::LABEL`，导致 "batch label not found"。已修复。
5. **app.asar 被占用**: electron-builder 打包时 app.asar 被其他进程锁定。kill EasyAgent/electron 进程后重试成功。
6. **CI frozen-lockfile 失败**: pnpm-lock.yaml specifiers 与 package.json 不同步。删除 lockfile 后重新 `pnpm install` 生成，`--frozen-lockfile` 验证通过。Commit `d5b71b3`。
7. **CI 根因修复 — windows-latest → windows-2022**: GitHub Actions `windows-latest` 已升级为 Server 2025 + VS 2026，node-gyp v10.3.1 不支持 VS 2026，导致 `better-sqlite3` 编译失败、CI 显示 0 jobs。修复：`ci.yml` 5个job + `release.yml` 3个job 全部固定为 `windows-2022` 保持 VS 2022 编译环境。同时修复 YAML 格式（流序列 `[a, b]` → 块序列）。CI #36 最终结果：**5/5 ✅ 全部通过** (Test Core, Test Server, Build Verification, Pipeline Tests, Coverage Gate)。Commits: `5ab6ef6`, `4324ef2`, `5625e7b`。
8. **CI 补全 Desktop 测试**: CI 缺少 Desktop test job (7文件/118测试)。修复: `ci.yml` 新增 `test-desktop` job (需先 build core+frontend)，`release.yml` test job 补全 desktop 测试。CI #37 6/6 jobs 出现，Test Desktop ✅ 通过，但 Coverage Gate 因 `@vitest/coverage-v8` 在 desktop 缺失报错 + Desktop coverage 总是 0%（渲染代码在 packages/frontend）。修复: 添加 `@vitest/coverage-v8` 到 desktop devDeps，Coverage Gate 仅覆盖 Core+Server。Commits: `dae293c`, `f556e92`, `46c8c1f`。
9. **测试文档同步修正**: 修正 `03_测试案例文档.md` 与实际代码差异 — ChatView.test.tsx→chatStore.test.ts，Sidebar.test.tsx 标记为计划中，TabBar 计数 9→8，总数 893→1052。
