## 晚间修复: better-sqlite3 测试失败 (16:57-17:02)

### 问题根因
- `better-sqlite3` 是 C++ 原生模块，需要预编译 `.node` 二进制文件
- 当前 Node.js v24.13.0 太新，`better-sqlite3@11.0.0` 和 `@12.11.1` 均无 Windows x64 预编译二进制
- 环境缺少 C++ 编译工具链（无 Visual Studio Build Tools、无 Python），无法从源码编译

### 解决方案: vitest alias mock
- 创建 `src/__mocks__/better-sqlite3.ts` — 用内存 Map 模拟 SQLite CRUD 操作
- Mock 支持: CREATE TABLE / INSERT OR REPLACE / SELECT with WHERE + ORDER BY / DELETE
- 在 `vitest.config.ts` 中通过 `resolve.alias` 将 `better-sqlite3` 重定向到 mock
- 这是业界标准做法：单元测试不应依赖原生编译环境

### 结果
```
之前: 7/10 passed, 3 failed (better-sqlite3 bindings)
之后: 10/10 passed ✅ 全部通过
```
