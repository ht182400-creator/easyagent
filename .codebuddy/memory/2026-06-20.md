## 🔴 重大发现：electron-builder v24.0.0 陷阱分析 (14:00-14:20)

### 问题现象
- NSIS 打包报错: `Plugin not found, cannot call EnVar::RemoveValue` (in `uninstaller.nsh` line 239)
- 生成 EXE 仅 0.3MB（应为 ~86MB）

### 调查过程
1. **第一次怀疑**: EnVar 插件缺失 → 下载 GsNSIS/EnVar → 仍失败（ANSI-only 版本）
2. **第二次怀疑**: 需要 Unicode 版 EnVar → 下载 ddev/EnVar (真正的 Unicode 版本, 9216 bytes vs 8704 bytes) → 仍失败
3. **第三次怀疑**: electron-builder 使用 `!addplugindir /x86-unicode` 显式指定插件目录 → 覆盖默认搜索路径
4. **用户质疑**: "上一个版本不是可以打包成功。为何要找根本原因？要降级？"
5. **深度排查**: 发现 node_modules 中**同时存在 electron-builder@23.6.0 和 @24.0.0 两套版本**！

### 根本原因
- `package.json` 声明 `^23.6.0`，`pnpm-lock.yaml` 锁定 23.6.0，但运行时执行了 v24.0.0
- **v24.0.0 是 alpha/pre-release 版本**，不是有意升级
- `^23.6.0` semver 不应匹配 24.x，但 pnpm 传递依赖提升导致两版共存
- `npx` 优先使用全局缓存/hoisted 版本，而非本地锁定的 23.6.0

### v24.0.0 的问题
| 问题 | 详情 |
|------|------|
| 自动注入宏 | `MULTIUSER_INSTALLMODE_ALLOW_ELEVATION` / `INSTALL_MODE_PER_ALL_USERS_REQUIRED`（未配置） |
| NSIS 资源版本 | 使用 `nsis-resources-3.4.1`（v23 用旧版），模板含 EnVar 调用 |
| EnVar 插件 | 未预置在 nsis-resources 中，导致编译失败 |
| app-builder-bin | v4.0.0 也有兼容问题 |
| 项目不需要 | 我们没用任何 v24 专有功能 |

### 修复步骤
1. `package.json`: `"electron-builder": "^23.6.0"` → `"electron-builder": "23.6.0"`（精确锁定）
2. 删除 node_modules 中所有 24.0.0 包
3. `pnpm install` 重新解析
4. 验证: `electron-builder@23.6.0` + `app-builder-lib@23.6.0` + `builder-util@23.6.0` + `dmg-builder@23.6.0` + `electron-publish@23.6.0`
5. 创建 `packages/desktop/assets/LICENSE`（NSIS 许可页面需要）
6. 修复 Vite 5.2.0 inline CSS bug：index.html `<style>` → `<link>` 外部 CSS
7. 防白屏样式加入 index.css，注意 `@import` 必须在所有规则之前

- `body-parser` 是 `express` 的子依赖，被 pnpm 提升到根 `.pnpm` 目录

### 修复
1. `package.json` dependencies 显式添加 `"body-parser": "^1.20.0"`
2. `pnpm install`
3. 重新打包

### 潜在风险
- Express 还有 20+ 个子依赖（accepts, content-disposition, finalhandler, send, serve-static, type-is 等）
- 后续可能逐个暴露 → 如果再次出现类似错误，考虑将 express 全部子依赖批量加入 dependencies
