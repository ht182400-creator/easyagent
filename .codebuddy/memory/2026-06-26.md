## [F10] Desktop "无法连接到后端服务" — CORS origin 拒绝 file:// 协议 (01:41 → 01:50 改进)

  2. 三元运算符 `corsOrigin ? corsOrigin : callback` 有歧义——CORS_ORIGIN 可能被 Electron 环境继承或设置
- **最终修复 (01:50)**:
  1. `packages/server/src/index.ts:426` — **完全移除三元运算符**，始终使用回调函数；若 `CORS_ORIGIN` 设定了则精确匹配，否则允许 file://、localhost、127.0.0.1

  3. `npx @electron/asar extract` 解包 `app.asar` → 替换 server dist → `npx @electron/asar pack` 重新打包 (mtime: 01:47)

- **后端验证**: EasyAgent.exe 启动后端口3456监听正常、/api/health 200 OK、/api/status 200 OK → 后端完全正常
- **真正根因**: 
  1. `config.tsx` 中 `ConfigProvider` 用 `useEffect` 同步 `apiBase`/`isDesktop` 到 `request.ts` 模块级变量
  2. `App.tsx` 的第一个 `useEffect` 立即调用 `loadSettings()` → `apiRequest('/api/config')`
  3. React 18 中父/子 `useEffect` 执行顺序不确定 → `_apiBase` 可能仍为空字符串
  4. 空 `_apiBase` 导致 fetch URL 为 `'' + '/api/config'` = 相对路径 `/api/config`
  5. Electron 从 `file://` 加载 → 解析为 `file:///api/config` → 必定失败
- **修复**:
  1. `config.tsx:8,59` — `useEffect` → `useLayoutEffect`（保证在子组件 effect 之前同步执行）
  2. `Dashboard.tsx:14-15,66-74,77-84` — 裸 `fetch('/api/status')` 和 `fetch('/api/config/templates')` 改为 `apiRequest('/api/status')` 和 `apiRequest('/api/config/templates')`（使用正确的 apiBase）
- **验证**: 重新编译打包后 `index-BBA_WhUs.js` (MD5: AD057167)，全 API 200 OK，端口监听正常
- **教训**: React `useEffect` 执行顺序在父子组件间不可靠，对外部模块级变量的同步必须用 `useLayoutEffect`；Dashboard 的裸 `fetch` 在 desktop file:// 协议下解析错误
