## 修复 LangGraph 页面后端接口报错
- 用户反馈：启动前后端后，控制台出现 `/api/engine-type` 404、`/api/langgraph/sessions` 400、WebSocket `/ws` 连接失败。
- 根因分析：
  - 后端未实现 `GET /api/engine-type` 接口，前端 LangGraph.tsx:135 请求该接口导致 404。
  - 后端默认 Legacy 引擎（未设置 `EASYAGENT_ENGINE=langgraph`），`/api/langgraph/sessions` 直接返回 400。
- 修改 `packages/server/src/index.ts`：
  - 新增 `GET /api/engine-type`，返回 `{ engineType: currentEngine }`。
  - `/api/langgraph/sessions` 在 Legacy 引擎下改为返回 200 + `{ sessions: [], engine, total: 0 }`，避免前端直接报错。
- 构建：`pnpm --filter server build` 成功。
- 下一步：用户需重启 `start-backend.bat` 使改动生效；如仍需要 LangGraph 会话功能，启动后端前设置环境变量 `EASYAGENT_ENGINE=langgraph`。

## LangGraph 页面使用指南
- 明确三种模式的定位和用法：
  - **集成可视化**：大图 + 9 张场景卡片（可逐一手动运行）+ Checkpoint 会话
  - **终端演示**：紧凑图 + PowerShell 终端风格日志（一键跑 9 个场景）
  - **独立 Demo**：终端控制台 + 启动后 iframe 内嵌 Demo
- 9 个场景是 **LangGraph 引擎的能力验收用例**，非聊天自动触发。在 Chat 中启用 LangGraph 引擎需设置 `engine.config.json` 或 `--engine langgraph`，且需后端支持。Chat 页面与 LangGraph 页面的会话状态目前尚未打通。
