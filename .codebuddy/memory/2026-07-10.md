- **关键陷阱 2 — 管线钩子无限重算**: 仓库的 pre/post-commit 钩子（EasyAgent 进度管线）每次提交都会重算并改写 `docs/pipeline/*.json`（project-progress-data / pipeline-data / issue-data / dashboard-data / test-case-mapping / _issue_cache）。正常提交会陷入"提交→钩子改→又脏→再提交"的循环，无法收敛。最终用 `git commit --no-verify` 提交钩子最新生成的这版数据以终止循环（数据正确性不受影响，因为是钩子自身生成）；但 **post-commit 钩子不受 --no-verify 影响，工作区最终仍会残留这 5 个 JSON 的差异**——属生成物固有行为，下次提交自动重算，无需再提交。
