## Logger UTF-8 乱码根本性修复 (08:39)
- 用户反馈服务日志中所有中文都是乱码（如"宸ュ叿宸叉敞鍐?" 应为 "工具已注册"）
- **根因**: pino v9 transport 使用 worker 线程，Windows 下 worker 线程的 stderr 编码不是 UTF-8，中文被 GBK 误解码
- **修复方案**: Windows 平台不再使用 pino-pretty worker transport，改为主线程同步 Transform stream
  - `logger.ts` 重写 `createLogger()`: `process.platform === 'win32'` 时调用 `createPrettyTarget()`
  - `createPrettyTarget()`: 用 `require.resolve('pino-pretty')` 找到模块，`pretty(opts)` 返回 Transform，直接作为 pino 的目标流
  - 非 Windows 平台保持原有 transport 模式不变
- **启动命令也修复**: PowerShell 窗口默认添加 `[Console]::OutputEncoding = UTF8`
- **修改文件**: `packages/core/src/utils/logger.ts` (核心修改)
- 服务重启验证正常，中文日志应可正常显示
