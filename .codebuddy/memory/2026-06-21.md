## 修复 Sessions 页面显示"0个会话"问题 - 2026-06-21 02:27

### 问题
Dashboard 显示"45个活跃会话"，但点击跳转到会话管理页面时显示"0个会话"。

### 根因
`packages/desktop/src/renderer/pages/Sessions.tsx` 第30-31行：
```javascript
apiFetch('/api/sessions')
  .then(r => r.json())  // ❌ 错误！apiFetch 已内置 res.json()，返回的是解析后的数据
```

`apiFetch` 函数（`api.ts` 第34-39行）已在内部调用 `res.json()` 返回解析后的数据（数组），
再对数组调用 `.json()` 方法会抛出 `TypeError`（数组没有 `.json()` 方法），
被 `.catch(() => setLoading(false))` 静默吞掉，sessions 保持空数组 `[]`。

**对比**：
- `/api/status` → Dashboard：返回 `{ sessionCount: 45 }`，Dashboard 正确显示
- `/api/sessions` → Sessions 页面：返回 `[...]`（会话数组），但前端解析失败

**Web 版不受影响**：web/Sessions.tsx 使用原生 `fetch()` 而非 `apiFetch`，`.then(r => r.json())` 是正确的。

### 修复
`desktop/Sessions.tsx`：移除 `.then(r => r.json())`，直接使用 apiFetch 的返回数据：
```javascript
apiFetch<Session[]>('/api/sessions')
  .then((data) => {
    setSessions(Array.isArray(data) ? data : []);
    setLoading(false);
  })
```

## Web vs Desktop 差异分析与约束规则 - 2026-06-21 02:39

### 为什么同样的问题 Web 不出现

**Web 和 Desktop 是完全独立的两套前端代码**：
- `packages/web/src/` — Web 版（65 个源文件）
- `packages/desktop/src/renderer/` — Desktop 版（65 个源文件）
- 各自拥有独立的 `pages/`、`components/`、`stores/` 副本，**不共享任何前端代码**

| 差异维度 | Web | Desktop |
|----------|-----|---------|
| 路由方案 | `BrowserRouter`（路径 `/sessions`） | `HashRouter`（路径 `/#/sessions`） |
| HTTP 库 | 原生 `fetch()`（需手动 `.json()`） | `apiFetch()` 封装（已内置 `.json()`） |
| 加载协议 | HTTP/HTTPS（Vite dev server） | `file://` + `http://127.0.0.1:3456` |

**Bug 1 黑屏** [🖥️ Desktop专属]：Web 使用 BrowserRouter，`<a href="/sessions">` 触发全页刷新到 `/sessions`，但 Vite dev server 有 SPA fallback 返回 `index.html`，React Router 接管后正常渲染。Desktop 使用 HashRouter + `file://` 协议，`<a href="/sessions">` 尝试访问 `file:///sessions`（不存在的文件路径）→ 黑屏。

**Bug 2 双重 .json()** [🖥️ Desktop专属]：Desktop 的 apiFetch 内部已调 `res.json()` 返回解析后对象，再调 `.json()` 导致 TypeError 被静默吞掉。Web 使用原生 `fetch()` 返回 Response 对象，`.json()` 是正确且必要的操作。

## 修复工具系统页面"HTTP undefined"错误 [🖥️ Desktop专属] - 2026-06-21 02:47

### 问题
工具系统页面显示 "加载失败: HTTP undefined"

### 根因
`Tools.tsx` 第83-86行使用 `apiFetch` 但当作原生 `fetch` 处理：
```javascript
apiFetch('/api/tools')
  .then(async (r) => {
    if (!r.ok) throw new Error(`HTTP ${r.status}`);  // r 是数组，无 .ok
    return r.json();  // 数组无 .json() 方法
  })
```
`apiFetch` 内部已调 `res.json()` 返回解析后的 `ToolInfo[]`，而代码把 `r` 当作 Response 对象检查 `.ok` 和 `.json()`。`r.ok` → `undefined`（falsy）→ `Error('HTTP undefined')`。

**Web 不受影响**：Web 版 `Tools.tsx` 使用原生 `fetch()` → 返回 Response → `.ok` 和 `.json()` 正确。

**教训**：同一 .tsx 文件 Web 使用原版 `fetch()`，Desktop 用 `apiFetch()` 替代，但调用方式不能照搬。

### 修复
- `apiFetch('/api/tools').then(r => { if (!r.ok) ... return r.json() })` → `apiFetch<ToolInfo[]>('/api/tools').then(data => setTools(Array.isArray(data) ? data : []))`
- 添加 `console.error` 错误日志

### 打包
- EasyAgent.exe: 168.9 MB (02:47:17)
- 需要强制清理 release 目录（旧 EasyAgent.exe 进程锁定文件）

### MEMORY.md 新增约束
- 新增 `Web ↔ Desktop 代码隔离约束` 章节
