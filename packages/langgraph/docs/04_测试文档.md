## 三、Mock 组件工厂

### 3.1 Mock Chat — 模拟 LLM 响应

```typescript
/**
 * 创建模拟的 chat 函数
 * 按调用索引依次返回预设响应
 *
 * @param responses - 预设响应数组，按调用顺序
 */
function createMockChat(responses: Array<{
  content?: string;                                        // 文本内容
  toolCalls?: Array<{                                      // 工具调用列表
    id: string;
    name: string;
    args: Record<string, unknown>;
  }>;
  finishReason?: string;                                   // 'stop' | 'tool_calls' | 'length'
}>) {
  let callIndex = 0;
  return vi.fn((_messages: unknown[], _options?: unknown) => {
    const response = responses[callIndex++] || {
      content: '默认响应',
      finishReason: 'stop',
    };
    return Promise.resolve({
      content: response.content || '',
      toolCalls: response.toolCalls?.map((tc) => ({
        id: tc.id,
        type: 'function' as const,
        function: { name: tc.name, arguments: JSON.stringify(tc.args) },
      })),
      finishReason: response.finishReason || 'stop',
      usage: { inputTokens: 10, outputTokens: 5 },
    });
  });
}
```

**使用示例:**

```typescript
// 单轮纯文本对话
const chat = createMockChat([
  { content: '你好！', finishReason: 'stop' },
]);

// 工具调用 + 总结
const chat = createMockChat([
  {
    content: '',
    toolCalls: [{ id: 'c1', name: 'calculate', args: { expr: '1+1' } }],
    finishReason: 'tool_calls',
  },
  { content: '结果是 2', finishReason: 'stop' },
]);

// 模拟错误
const chat = vi.fn(() => Promise.reject(new Error('API 超时')));
```

### 3.2 Mock Tool Executor — 模拟工具执行

```typescript
/**
 * 创建模拟的工具执行器
 *
 * @param results - 工具名 → 执行结果的映射
 */
function createMockToolExecutor(
  results: Record<string, { success: boolean; content: string }> = {}
) {
  return {
    execute: vi.fn((name: string, _params: Record<string, unknown>) => {
      const result = results[name] || {
        success: true,
        content: `工具 ${name} 执行成功`,
      };
      return Promise.resolve(result);
    }),
  };
}
```

**使用示例:**

```typescript
// 正常工具
const executor = createMockToolExecutor({
  get_weather: { success: true, content: '晴天' },
  get_time: { success: true, content: '12:00' },
});

// 工具失败
const executor = createMockToolExecutor({
  broken_tool: { success: false, content: '执行失败: 参数错误' },
});
```

**TC-2.6: run 后 resume 同一会话**

```typescript
  agent = buildAgent([
    { content: '第一轮回答', finishReason: 'stop' },
  ]).agent;

  const r1 = await agent.run('第一轮', { sessionId: 'resume-test' });
  expect(r1.response).toBe('第一轮回答');

  const r2 = await agent.resume('resume-test', '继续对话');
  expect(r2.sessionId).toBe('resume-test');
  expect(r2.response.length).toBeGreaterThan(0);
});
```

**resume 内部流程:**

```
1. checkpointer.getLatestState(sessionId)
   → 从 SQLite 读取最新的 checkpoint
2. 如果提供了新消息，追加到 messages 数组
3. graph.invoke(input, { configurable: { thread_id: sessionId } })
   → LangGraph 自动从 checkpoint 恢复状态
4. 返回 AgentResult
```

## 八、已知限制与注意事项

### 8.1 Mock 并非真实 LLM 行为

- Mock Chat 按索引返回预设响应，不真正解析 message 历史
- 工具定义通过 `getToolDefinitions` 注入，Mock 中不需要真实定义
- `finishReason` 必须手动设为 `'tool_calls'` 才会触发工具路由

### 8.2 流式测试限制

- 当前 Mock Chat 是非流式的，`stream()` 测试只能验证 `done` 事件
- 要测试完整流式行为（`thinking`/`tool_call`/`tool_result`/`response` 事件序列），
  需要实现 `stream()` 方法的 Mock ChatModel

### 8.3 Checkpoint 格式差异

- LangGraph 内部 checkpoint 格式为 `{ v, id, ts, channel_values, channel_versions, versions_seen }`
- `channel_values` 包含实际的 State（messages, turnCount 等）
- `getLatestState` 已适配自动提取 `channel_values`

### 8.4 SqliteCheckpointer 必须是 extends

```typescript
// ❌ 错误 — PregelLoop 调用 this.checkpointer.getNextVersion.bind(...) 时报 undefined
class SqliteCheckpointer implements BaseCheckpointSaver { ... }

// ✅ 正确 — 继承抽象类以获得 getNextVersion 等基类方法
class SqliteCheckpointer extends BaseCheckpointSaver { ... }
```

### 8.5 所有 Checkpointer 方法必须 async

LangGraph 的 `BaseCheckpointSaver` 定义的所有方法都返回 `Promise`:
- `getTuple(): Promise<CheckpointTuple | undefined>`
- `list(): AsyncGenerator<CheckpointTuple>`
- `put(): Promise<RunnableConfig>`
- `putWrites(): Promise<void>`

| Checkpointer | `数据库连接`, `getTuple`, `put (保存 checkpoint)`, `getLatestState`, `listThreads` |
| MemoryManager | `生成摘要` |
