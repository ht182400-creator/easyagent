# 整体架构设计

> 基于 LangGraph 官方推荐模式：StateGraph + Checkpointer + Memory

---

## 一、LangGraph 核心概念映射

```
StateGraph  = 声明式状态机
Node        = 处理函数 (State → Partial<State>)
Edge        = 状态转移 (Node → Next Node)
ConditionalEdge = 条件分支 (State → 路由决策 → Next Node)
Compile     = 生成可执行的 Runnable
Invoke      = 传入初始状态 → 执行直到终止
Checkpointer = 每步自动保存 State 快照
```

## 二、思考-行动-观察 环形图

```
                    ┌──────────────┐
                    │   __start__  │
                    └──────┬───────┘
                           │
                    ┌──────▼───────┐
              ┌─────│    think     │◄──────────────┐
              │     │  (LLM 调用)   │               │
              │     └──────┬───────┘               │
              │            │                       │
              │     ┌──────▼───────┐               │
              │     │ routeAfter   │               │
              │     │   Think      │               │
              │     └──┬───────┬───┘               │
              │        │       │                    │
              │   tool_calls  no_tools              │
              │        │       │                    │
              │  ┌─────▼──┐    │                    │
              │  │  act   │    │                    │
              │  │(工具执行)│   │                    │
              │  └───┬────┘    │                    │
              │      │         │                    │
              │ ┌────▼────┐    │                    │
              │ │ observe │    │                    │
              │ │(结果封装)│   │                    │
              │ └───┬─────┘   │                    │
              │     │         │                    │
              └─────┘         │                    │
                              │                    │
                       ┌──────▼───────┐            │
                       │   __end__    │            │
                       └──────────────┘            │
                                                   │
      think → routeAfterThink ──(tool_calls)──► act
         ↑                                       │
         └────────── observe ◄───────────────────┘
```

**流程说明**：

1. **think** — 构建消息列表 → 调用 LLM → 返回 AIMessage（可能包含 tool_calls）
2. **routeAfterThink** — 检查 AIMessage 是否有 tool_calls → 路由到 act 或 END
3. **act** — 解析 tool_calls → 并行/串行执行工具 → 返回 ToolMessage[]
4. **observe** — 将工具结果封装 → 追加到消息历史 → 回到 think

## 三、状态定义 (AgentState)

```typescript
// 继承 LangGraph 的 MessagesState（内置 messages 管理）
interface AgentState extends MessagesState {
  // === 控制字段 ===
  turnCount: number;           // 当前轮次
  maxTurns: number;            // 最大轮次 (默认 25)
  shouldContinue: boolean;     // 是否继续循环

  // === 工具相关 ===
  lastToolCalls: ToolCall[];   // 最近一次工具调用
  lastToolResults: ToolResult[]; // 最近一次工具结果

  // === 用量追踪 ===
  totalInputTokens: number;
  totalOutputTokens: number;

  // === 会话信息 ===
  sessionId: string;
  workspace: string;

  // === 中断/恢复 ===
  interruptReason?: string;    // 中断原因（人工审批等）
  resumedFromCheckpoint?: string; // 恢复的检查点 ID
}
```

## 四、节点详述

### think — LLM 思考节点

```
输入: AgentState (含 messages, systemPrompt)
处理:
  1. 构建消息列表 [SystemMessage, ...历史消息]
  2. 获取工具定义 (ToolRegistry.getDefinitions())
  3. 调用 adapter.chat(messages, { tools, stream? })
  4. 解析 AIMessage (content + tool_calls)
输出: { messages: [AIMessage], turnCount: +1, totalTokens: +usage }
```

### act — 工具执行节点

```
输入: AgentState (含 lastToolCalls, ToolRegistry)
处理:
  1. 遍历 tool_calls
  2. 并行调用 ToolRegistry.execute(name, params, context)
  3. 收集 ToolResult[]
输出: { lastToolResults: ToolResult[] }
```

### observe — 观察与封装节点

```
输入: AgentState (含 lastToolResults)
处理:
  1. 将 ToolResult[] 转换为 ToolMessage[]
  2. 追加到 messages 列表
  3. 清除 lastToolCalls / lastToolResults
输出: { messages: [ToolMessage, ...] }
```

## 五、Checkpoint 持久化架构

```
每次 SuperStep (node执行+状态更新) 后自动保存:

┌─────────────┐     ┌──────────────────┐
│  LangGraph   │────▶│ SqliteCheckpoint │
│  StateGraph  │     │     Saver        │
└─────────────┘     └────────┬─────────┘
                             │
                    ┌────────▼─────────┐
                    │  .checkpoints.db │
                    │  (SQLite)        │
                    │  ┌────────────┐  │
                    │  │ checkpoints │  │
                    │  │ writes      │  │
                    │  └────────────┘  │
                    └─────────────────┘

checkpoint 表结构:
  - thread_id    (会话 ID)
  - checkpoint_id (检查点 UUID)
  - parent_id    (父检查点 UUID)
  - checkpoint   (完整 State JSON)
  - metadata     (step, source, timestamp)
  - created_at
```

## 六、记忆系统架构

```
Memory 分层:
┌────────────────────────────────────────┐
│  Layer 3: 长期记忆 (Semantic Memory)   │
│  - 向量存储关键信息                     │
│  - 用户偏好、项目知识                   │
│  - (预留接口，Phase 3 实现)             │
├────────────────────────────────────────┤
│  Layer 2: 会话记忆 (Session Memory)    │
│  - messages 完整历史（Checkpoint 管理） │
│  - 摘要压缩 (当消息数 > 阈值)           │
├────────────────────────────────────────┤
│  Layer 1: 工作记忆 (Working Memory)    │
│  - 当前 State 图执行上下文              │
│  - turnCount, lastToolCalls 等         │
└────────────────────────────────────────┘
```

## 七、对外接口 (Agent 类)

```typescript
class LangGraphAgent {
  // === 构造 ===
  constructor(config: {
    adapter: BaseAdapter;           // 复用现有适配器
    toolRegistry: ToolRegistry;     // 复用现有工具系统
    systemPrompt?: string;
    maxTurns?: number;
    checkpointer?: BaseCheckpointSaver;
    memory?: BaseMemory;
  });

  // === 核心方法（兼容 AgentEngine 接口）===
  async run(userMessage: string, options?: {
    sessionId?: string;
    signal?: AbortSignal;
  }): Promise<AgentResult>;

  // === 流式运行 ===
  async *stream(userMessage: string, options?: {
    sessionId?: string;
  }): AsyncGenerator<AgentEvent>;

  // === Checkpoint 管理 ===
  async resume(sessionId: string): Promise<AgentResult>;
  async getState(sessionId: string): Promise<AgentState>;
  async listSessions(): Promise<SessionSummary[]>;

  // === 控制 ===
  abort(): void;
  clearHistory(sessionId: string): void;
}
```
