## 二、7 状态机定义

### 2.1 状态转移表

```
                    触发事件
状态 ────────────────────────────────────────────→ 新状态

idle:
  ├── startAutoCheck()          → checking
  ├── check-update IPC          → checking
  └── (无更新)                  → idle (保持)

checking:
  ├── update-available 事件     → available
  ├── update-not-available      → idle
  └── error 事件                → error

available:
  ├── autoDownload 开始         → downloading
  ├── download-progress 事件    → downloading
  └── error 事件                → error

downloading:
  ├── download-progress 事件    → downloading (更新进度)
  ├── update-downloaded 事件    → downloaded
  └── error 事件                → error

downloaded:
  ├── install-update IPC        → installing (quitAndInstall)
  └── dialog "稍后"             → downloaded (保持)

error:
  ├── check-update IPC (重试)   → checking
  └── (网络恢复后自动重试)       → idle → checking

installing:
  └── quitAndInstall()          → 应用退出
```

### 2.2 状态定义表

| 状态 | 含义 | 触发方 | 允许的用户操作 | 防重复策略 |
|------|------|--------|---------------|-----------|
| `idle` | 无更新活动 | 初始/无更新 | 点击"检查更新" | — |
| `checking` | 正在检查更新 | auto/manual | 等待（显示"检查中"） | 重复点击返回 "正在检查中" |
| `available` | 发现新版本 | electron-updater | 等待（自动开始下载） | 返回当前状态 |
| `downloading` | 下载中 | electron-updater | 查看进度 | 返回当前进度 |
| `downloaded` | 下载完成 | electron-updater | **立即重启安装** | 返回安装提示（不重复下载） |
| `error` | 检查/下载失败 | electron-updater | **重试** | 清除错误后重新开始 |
| `installing` | 安装中 | 用户 | 无（应用即将退出） | 返回"安装中" |

## 三、IPC 通信全矩阵

### 3.1 IPC 通道定义

| # | 通道名 | 方向 | 类型 | 输入参数 | 输出 |
|---|--------|------|------|---------|------|
| 1 | `check-update` | renderer→main | invoke | 无 | `{success, status, version, lastUpdateInfo}` |
| 2 | `get-update-status` | renderer→main | invoke | 无 | `{isUpdateSupported, currentVersion, lastUpdateStatus, lastUpdateInfo}` |
| 3 | `install-update` | renderer→main | invoke | 无 | `{success, status}` |
| 4 | `simulate-update` | renderer→main | invoke | `{scenario, version?}` | `{success, message}` |
| 5 | `update-status` | main→renderer | push event | 状态对象 | 通过 `onUpdateStatus` 回调接收 |

### 3.2 `check-update` IPC 处理矩阵（6 大场景）

| 场景 | 前置 `lastUpdateStatus` | 处理逻辑 | 返回 | 前端效果 |
|------|------------------------|---------|------|---------|
| **S1** 已下载 | `downloaded` | 推送 `downloaded` 事件，不重复下载 | `{success:true, status:'downloaded', version}` | 显示安装按钮 |
| **S2** 检查中 | `checking` | 防重复，直接返回 | `{success:true, status:'checking', message}` | 显示"请稍候" |
| **S3** 已发现 | `available` | autoDownload 即将触发 | `{success:true, status:'available', version}` | 显示"下载即将开始" |
| **S4** 下载中 | `downloading` | 推送当前进度 | `{success:true, status:'downloading', percent}` | 显示进度条 |
| **S5** 出错后重试 | `error` | 清除错误 → 检查 → 下载 | 走正常流程 | 重新开始检查 |
| **S6** 空闲开始 | `idle` | `checkForUpdates()` → `downloadUpdate()` | `{success:true, status, version}` | 完整流程 |

### 3.3 `update-status` 推送事件格式

```typescript
// electron-updater 事件 → IPC 推送格式
{
  status: 'idle' | 'checking' | 'available' | 'downloading' | 'downloaded' | 'error' | 'installing',
  version?: string,        // available / downloaded 时携带
  releaseDate?: string,    // available 时携带
  percent?: number,        // downloading 时携带
  message?: string,        // error 时携带
  transferred?: number,    // downloading 时携带（模拟测试）
  total?: number,          // downloading 时携带（模拟测试）
}
```

### 3.4 `install-update` IPC 处理

| 前置状态 | 处理 | 返回 |
|---------|------|------|
| `downloaded` | `lastUpdateStatus = 'installing'` → `quitAndInstall()` | `{success:true, status:'installing'}` |
| 其他状态 | 拒绝 | `{success:false, error:'没有可安装的更新'}` |
| 未启用 | 拒绝 | `{success:false, error:'自动更新未启用'}` |

### 4.2 场景详细流程

#### T1: 正常完整流程
```
idle → [用户点击"检查更新"]
  → Step 0: 查询 getUpdateStatus() → idle
  → Step 1: 查询 Server API /api/version/check → hasUpdate=true
  → Step 2: IPC check-update → checking
  → electron-updater: checkForUpdates() → 发现新版本
  → update-available 事件 → available
  → autoDownload → downloading (进度0%→100%)
  → update-downloaded 事件 → downloaded
  → 系统弹窗: "是否立即重启？"
  → 用户点"立即重启" → installing → quitAndInstall()
```

#### T2: 后台下载中手动检查
```
downloading (后台自动下载到45%)
  → [用户进入设置页]
  → useEffect: getUpdateStatus() → downloading, percent=45
  → 显示进度条45%
  → [用户点击"检查更新"]
  → Step 0: getUpdateStatus() → downloading
  → setUpdaterStatus({ status:'downloading', percent:45 })
  → Step 1: Server API → hasUpdate=true
  → Step 2: IPC check-update → 主进程检测到 downloading
  → 返回 { success:true, status:'downloading', lastUpdateInfo:{percent:45} }
  → updaterStatus 保持 downloading，进度条继续更新
```

#### T5: 下载失败后重试
```
error (ETIMEDOUT)
  → UI: 黄色 error 卡片 + "重试"按钮
  → [用户点击"重试"]
  → checkForUpdates() 被调用
  → Step 0: getUpdateStatus() → error
  → Step 2: IPC check-update → 主进程检测到 error
  → 清除 error → 开始 checkForUpdates()
  → 重新走完整流程
```

## 六、架构决策记录 (ADR)

### ADR-001: 避免 `checkForUpdatesAndNotify()` 重复调用冲突

**问题**: v0.5.17 中自动检查(10s后)和手动检查都调用 `checkForUpdatesAndNotify()`，该函数内部有防重复机制，导致手动点击时被静默跳过。

**决策**: 自动检查继续使用 `checkForUpdatesAndNotify()`(带自动下载)，手动检查使用 `checkForUpdates()` + 显式 `downloadUpdate()`。两者通过 `lastUpdateStatus` 状态机互斥。

**影响**: 手动和自动更新通道相互独立，但共享同一个状态机变量，通过状态检查避免冲突。

---

### ADR-002: "稍后提醒"后保留安装入口

**问题**: 自动下载完成后弹出系统对话框，用户点"稍后提醒"后，如果无法再次找到安装入口，体验很差。

**决策**: 
1. 弹窗点"稍后"后状态保持 `downloaded`
2. 设置页始终显示"立即重启安装"按钮
3. 用户进入设置页时通过 `getUpdateStatus()` 查询到 `downloaded` 状态即显示按钮

**影响**: 用户无论何时进入设置页，都能看到可安装的更新。

---

### ADR-003: 错误状态保留版本信息

**问题**: 下载中断后，用户想知道"正在下载哪个版本时失败了"。

**决策**: `lastUpdateInfo` 在 error 状态时保留已获取的 `version` 字段，仅追加 `error` 消息。

**影响**: 前端可显示更详细的错误信息，如"正在下载 v0.5.20 时失败"。

---

### ADR-004: 前端辅助函数置于组件外

**问题**: 6 个状态渲染辅助函数在组件内定义会导致每次渲染重新创建。

**决策**: `getStatusCardStyle`、`getStatusIcon`、`getStatusTitle`、`getStatusDescription`、`getStatusAction` 全部定义为组件外的纯函数。

**影响**: 零重渲染开销，函数只依赖参数不依赖组件状态。

---

### ADR-005: 状态推送 + 主动查询双通道

**问题**: 设置页面挂载时，自动更新可能已经完成。仅依赖事件监听会错过已发生的事件。

**决策**: 
1. `useEffect` 挂载时调用 `getUpdateStatus()` 查询当前状态
2. 同时注册 `onUpdateStatus()` 监听后续事件
3. useEffect 返回清理函数移除监听器

**影响**: 无论设置页面何时打开，都能获取正确的更新状态。

## 七、代码结构

### 7.1 文件职责

```
packages/
├── desktop/src/
│   ├── main.ts                    # 主进程: 状态机 + IPC handlers + electron-updater
│   ├── preload.ts                 # 预加载: contextBridge API 暴露
│   └── renderer/... (不使用)
└── frontend/src/
    └── pages/Settings.tsx         # 设置页: 更新状态显示 + 用户操作按钮
```

### 7.2 数据流

```
electron-updater 事件
  │
  ├── update-available    → lastUpdateStatus = 'available'  → IPC push
  ├── download-progress   → lastUpdateStatus = 'downloading' → IPC push
  ├── update-downloaded   → lastUpdateStatus = 'downloaded'  → IPC push + dialog
  └── error               → lastUpdateStatus = 'error'       → IPC push

IPC check-update handler
  │
  ├── 读取 lastUpdateStatus
  ├── 根据状态决定行为
  └── 返回 { success, status, version, lastUpdateInfo }

Settings.tsx
  │
  ├── useEffect 挂载 → getUpdateStatus() → 初始状态
  ├── onUpdateStatus 事件 → setUpdaterStatus() → 重新渲染
  ├── checkForUpdates() → getUpdateStatus → Server API → IPC check-update
  └── handleInstallUpdate() → IPC install-update → quitAndInstall()
```

## 八、性能考量

| 考量点 | 策略 |
|--------|------|
| 状态推送频率 | downloading 期间 progress 事件可能高频触发，通过 React 状态更新自动批处理 |
| 防重复检查 | `lastUpdateStatus === 'checking'` 时拒绝新的检查请求 |
| 防重复下载 | `lastUpdateStatus === 'downloaded'` 时直接返回安装提示 |
| 内存泄漏 | useEffect 返回清理函数移除 IPC 事件监听 |
| 辅助函数重渲染 | 6 个纯函数定义在组件外，无闭包依赖 |
