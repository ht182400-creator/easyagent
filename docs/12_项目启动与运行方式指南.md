# EasyAgent 项目启动与运行方式指南

> 版本: v1.0 | 日期: 2026-06-26 | 基于 v0.5.5

---

## 一、三种运行模式

EasyAgent 支持三种运行方式，每种方式的后端/前端提供方式不同：

```
┌──────────────────────────────────────────────────────────────────┐
│                      运行模式全景图                               │
├──────────────┬─────────────────┬─────────────────────────────────┤
│    模式       │    后端 :3456    │      前端                       │
├──────────────┼─────────────────┼─────────────────────────────────┤
│  Web 版       │ start-backend   │ Vite dev server (:5173)        │
│              │ .bat 启动        │ 浏览器访问                      │
├──────────────┼─────────────────┼─────────────────────────────────┤
│  Desktop 版   │ EXE 内嵌启动     │ Electron 窗口 (loadFile)        │
│              │ (自动)           │ 预编译 dist/renderer            │
├──────────────┼─────────────────┼─────────────────────────────────┤
│  Ink CLI 版   │ CLI 内嵌         │ 终端 (React Ink)               │
└──────────────┴─────────────────┴─────────────────────────────────┘
```

## 二、Web 版启动 (浏览器)

### 2.1 完整启动流程

```bash
# 终端 1: 启动后端 (端口 3456)
start-backend.bat

# 终端 2: 启动前端 (端口 5173)
start-frontend.bat
```

然后浏览器打开 **http://localhost:5173**

### 2.2 架构图

```
浏览器 http://localhost:5173
    │
    ▼
Vite Dev Server (:5173)          ← start-frontend.bat
    │  /api → proxy → :3456
    │  /ws  → proxy → ws://:3456
    ▼
Express 后端 (:3456)              ← start-backend.bat
    │
    ▼
SQLite 数据库 + better-sqlite3
```

### 2.3 关键文件

| 文件 | 作用 |
|------|------|
| `start-backend.bat` | 调用 `sqlite3-loader.mjs` 切换 .node 版本 → 启动 `packages/server/dist/index.js` |
| `start-frontend.bat` | 在 `packages/web/` 下执行 `npx vite --port 5173` |
| `packages/web/vite.config.ts` | 配置 `/api` → `:3456` 代理、`/ws` WebSocket 代理 |

### 2.4 前提条件

- Node.js v24 (MODULE_VERSION=137)
- `better-sqlite3` 已为 Node v24 编译（`npx node-gyp rebuild`）
- 端口 3456、5173 未被占用

## 三、Desktop 版启动 (EXE)

### 3.1 启动方式

直接双击 `EasyAgent.exe`，无需任何命令行操作。

### 3.2 架构图

```
┌─────────────────────────────────────────┐
│            EasyAgent.exe                 │
│                                          │
│  ┌────────────────────────────────────┐ │
│  │       Electron 主进程               │ │
│  │  ┌──────────────────────────────┐  │ │
│  │  │  startBackendServer()        │  │ │
│  │  │  Express 后端 (:3456)         │  │ │
│  │  │  + better-sqlite3 (Electron  │  │ │
│  │  │    Node v20, MODULE 123)     │  │ │
│  │  └──────────────────────────────┘  │ │
│  └────────────────────────────────────┘ │
│                                          │
│  ┌────────────────────────────────────┐ │
│  │       Electron 渲染进程             │ │
│  │  loadFile('dist/renderer/index.    │ │
│  │  html') → React 前端               │ │
│  │  (预编译静态资源，非 dev server)     │ │
│  └────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

### 3.3 关键区别

| 特性 | Web 版 | Desktop 版 |
|------|--------|-----------|
| 后端启动 | 手动 `start-backend.bat` | EXE 自动内嵌启动 |
| 前端方式 | Vite dev server (热更新) | 预编译静态文件 (无热更新) |
| Node 版本 | 系统 Node v24 (MODULE 137) | Electron 内嵌 Node v20 (MODULE 123) |
| sqlite3 | `better_sqlite3_system.node` | `better_sqlite3_electron.node` |
| 前端端口 | `:5173` | 无端口 (loadFile 协议) |
| `file://` 协议 | 不涉及 | 是 (CORS 需特殊处理) |

## 四、组合运行方式

### 4.1 EXE + Web 前端 (最灵活)

**场景**: 开发前端时用 EXE 提供后端，避免手动管理后端进程。

```
EasyAgent.exe (已启动) → 后端 :3456 ✅
         +
start-frontend.bat     → Vite :5173 (代理到 EXE 的 :3456)
         =
浏览器 http://localhost:5173 → 热更新前端开发
```

**步骤**:
1. 双击 `EasyAgent.exe` 启动
2. 运行 `start-frontend.bat`
3. 浏览器打开 `http://localhost:5173`

**注意**: 不需要运行 `start-backend.bat`，否则会端口冲突 (3456 已被 EXE 占用)。

### 4.2 纯 Web (无 EXE)

```
start-backend.bat  → 后端 :3456
start-frontend.bat → 前端 :5173
浏览器 http://localhost:5173
```

### 4.3 Desktop + Web 前端同时

```
┌──────────────┐     ┌──────────────────┐
│ EasyAgent.exe│     │ 浏览器 :5173      │
│ 后端 :3456    │◄───│ (Vite 代理)       │
│ Electron窗口  │     │                  │
└──────────────┘     └──────────────────┘
      共享同一个后端 :3456
```

两者可以**同时使用**，共享同一个后端和数据。

## 五、端口与进程速查

| 端口 | 进程 | 启动方式 | 说明 |
|------|------|---------|------|
| 3456 | Express 后端 | `start-backend.bat` 或 EXE 内嵌 | API + WebSocket |
| 5173 | Vite dev server | `start-frontend.bat` | Web 前端开发服务器 |
| — | Electron 窗口 | `EasyAgent.exe` | Desktop 原生窗口 |

```bash

# 检查前端是否在运行

# 测试后端 API
curl http://localhost:3456/api/status

# 测试前端代理
curl http://localhost:5173/api/status
```

## 六、常见问题排查

### 6.1 `start-backend.bat` 启动失败

| 错误信息 | 原因 | 解决 |
|----------|------|------|
| `缺少预编译文件: better_sqlite3_system.node` | better-sqlite3 未编译或文件名不对 | `npx node-gyp rebuild` → 复制为 `_system.node` |
| `NODE_MODULE_VERSION mismatch` | .node 文件编译时的 Node 版本与当前不匹配 | 用当前 Node 版本重新 `node-gyp rebuild` |
| `端口 3456 已被占用` | 已有后端实例在运行 | 关闭其他 EasyAgent 进程或 `taskkill /F /IM node.exe` |

### 6.2 Web 前端显示"无法连接到后端"

1. 确认后端正在运行: `netstat -ano | findstr :3456`
2. 确认 Vite 代理正常: `curl http://localhost:5173/api/status`
3. 如果使用 EXE 提供后端，确认 EXE 已启动

### 6.3 EXE 显示"无法连接到后端服务"

1. 确认 `better-sqlite3_electron.node` 存在且为 Electron 30 (MODULE 123) 编译
2. 检查 `asarUnpack` 配置是否包含 `better-sqlite3/**`
3. 查看 Electron 控制台 (F12) 网络请求

## 七、better-sqlite3 双版本管理

项目需要两套 `better_sqlite3` 编译产物：

| 版本 | 文件名 | MODULE | 用途 | 大小 |
|------|--------|--------|------|------|
| 系统 Node | `better_sqlite3_system.node` | 137 | Web 版后端 | ~1.76 MB |
| Electron | `better_sqlite3_electron.node` | 123 | Desktop 内嵌后端 | ~1.75 MB |

### 编译命令

```bash
# 系统 Node 版本 (用于 Web)
cd node_modules/.pnpm/better-sqlite3@12.11.1/node_modules/better-sqlite3
npx node-gyp rebuild
copy build\Release\better_sqlite3.node build\Release\better_sqlite3_system.node

# Electron 版本 (用于 Desktop)
npx node-gyp rebuild --target=30.0.0 --arch=x64 --dist-url=https://electronjs.org/headers
copy build\Release\better_sqlite3.node build\Release\better_sqlite3_electron.node
```

### 缓存位置

```
.codebuddy/sqlite3-cache/
├── better_sqlite3_system.node    ← 系统 Node 缓存
└── better_sqlite3_electron.node  ← Electron 缓存
```

## 八、快速启动检查表

启动前按此顺序验证：

```
□ 1. Node 版本: node -v (应为 v24.x)
□ 2. sqlite3 加载: node -e "require('better-sqlite3')" 无报错
□ 3. 启动后端: start-backend.bat → 看到 "EasyAgent Server v0.5.5"
□ 4. 测试 API: curl http://localhost:3456/api/status → 200 OK
□ 5. 启动前端: start-frontend.bat → Vite ready
□ 6. 打开浏览器: http://localhost:5173
```
