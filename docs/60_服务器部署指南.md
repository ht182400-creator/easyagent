# EasyAgent 服务器部署指南（Windows 云服务器）

> 目标：把 EasyAgent 部署到 Windows 云服务器 `82.156.71.231`，对外可访问。
> **立即可用**：`http://82.156.71.231:3456`（非 80 端口，绕开备案拦截，见 §4.1）。
> **干净域名** `http://CCCN.fable5.icu` 需域名备案或 EdgeOne 加速（见 §4.2）；SSL 证书后续再加（见 §5）。
> 仓库：`https://github.com/ht182400-creator/easyagent.git`（公开，HTTPS 可直接 clone）

> ⚠️ **实战重要补充（必读）**：本指南的 §4/§5 描述的是「理想态」（域名直连 80 + Caddy 上 HTTPS）。
> 但 `fable5.icu` **未在工信部备案**，实测会触发两类拦截，原方案直接照搬会失败：
> 1. **80 端口备案页拦截**：未备案域名 + 大陆服务器公网 IP 的 80 被腾讯云网络层拦（返回"未完成备案"页，nginx 无日志）。
> 2. **443 SNI 拦截**：未备案域名的 HTTPS 在 TLS 握手阶段被腾讯云基于 SNI 切断，哪怕有真证书也到不了 nginx（IP 直连 443 不受 SNI 影响，可成功）。
>
> **当前已验证可用的兜底方案**：在轻量服务器「防火墙」放行 `TCP 3456` → 用 `http://82.156.71.231:3456` 访问（非 80/443 端口，绕开备案拦截）。
> 想要干净域名 + 无警告 HTTPS，需另购 **EdgeOne 含「站点加速」额度的付费套餐**（免费 Pages 套餐不含该额度，删 Pages 也腾不出）。
> 完整背景、实测数据、方案对比与踩坑经验见 👉 **`docs/61_部署讨论与结论复盘.md`**。

## 0. 部署架构（无需改动任何业务代码）

```
浏览器 → CCCN.fable5.icu (DNS A 记录 → 82.156.71.231:80)
              │  同源访问（不会触发 CORS 预检）
              ▼
   EasyAgent Server (Node, HOST=0.0.0.0, PORT=80)
     ├─ /         → 托管 packages/web/dist（静态页面）
     ├─ /api/*    → REST API
     └─ /ws       → WebSocket（流式对话）
```

**为什么这么简单**：Server 本身用 `express.static` 同时托管前端页面 + API + WebSocket；
前端默认 `apiBase:''`、`wsBase:'/ws'`（相对路径），同源部署时自动变为 `CCCN.fable5.icu/api`
与 `ws://CCCN.fable5.icu/ws`。所以**只需在服务器上跑一个 Node 进程**即可。

> 唯一坑：better-sqlite3 是原生模块，必须与服务器 Node 的 ABI 完全一致。
> 项目 `preinstall` 会**拦截 Node ≥ 24**，且 `pnpm install` 会自动编译该模块。
> 因此服务器必须用 **Node 18 / 20 / 22 LTS**（推荐 20 或 22 LTS）。

## 1. 前置环境准备（服务器上，仅需一次）

通过远程桌面（RDP）登录 `82.156.71.231`，用 **管理员 PowerShell** 执行：

### 1.1 安装 Git
```powershell
winget install Git.Git
```

### 1.2 安装 Node.js LTS（推荐 22 LTS，严禁 24）
```powershell
winget install OpenJS.NodeJS.LTS
# 安装后重新打开 PowerShell，确认版本在 18~22 之间
node -v
```
> 若已误装 Node 24：用 `winget uninstall OpenJS.NodeJS` 卸载，再装 LTS。

### 1.3 安装 pnpm
```powershell
npm install -g pnpm@11.7.0
pnpm -v
```

### 1.4 安装 C++ 编译工具链（better-sqlite3 必须）
better-sqlite3 在 Windows 上需从源码编译，依赖：
- **Visual Studio 2022 Build Tools** + “使用 C++ 的桌面开发” 工作负载
- **Python 3**（node-gyp 需要）

```powershell
# C++ 生成工具
winget install Microsoft.VisualStudio.2022.BuildTools --override "--add Microsoft.VisualStudio.Workload.VCTools;includeRecommended"
# Python
winget install Python.Python.3.12
```
安装后**重启 PowerShell**，确认：
```powershell
python --version
# 以及 cl 可用（VS 开发者命令提示符中）
```

## 2. 拉取代码并部署

### 2.0 最简方式（推荐）：一行命令全自动
在服务器**管理员 PowerShell** 里粘贴一行，脚本会自动装环境（Git/Node LTS/pnpm/VS 构建工具/Python）
→ 克隆 → 部署 → 启动：
```powershell
irm https://raw.githubusercontent.com/ht182400-creator/easyagent/main/scripts/bootstrap-server.ps1 | iex
```
> VS Build Tools 装完可能需重启；重启后再次粘贴同一行即可（已装项会自动跳过）。
> 脚本源码见 `scripts/bootstrap-server.ps1`，可加 `-CloneDir D:\other` 改路径。

### 2.1 手动方式：克隆仓库
```powershell
# 建议放到 D:\easyagent 或 C:\easyagent
cd D:\
git clone https://github.com/ht182400-creator/easyagent.git easyagent
cd easyagent
```

### 2.2 一键部署（校验环境 + 装依赖 + 编译 + 构建）
```powershell
pwsh scripts/deploy-server.ps1
```
脚本会自动：校验 Node 版本 → `git pull` → `pnpm install`（编译 better-sqlite3）→
激活系统 Node 版原生模块 → 构建 `core / langgraph / server / web`。

若想构建后直接启动，加 `-Start`：
```powershell
pwsh scripts/deploy-server.ps1 -Start
```

## 3. 启动服务

> ⚠️ **绑定 80 端口需要管理员权限**。请用“管理员 PowerShell / 右键以管理员身份运行”。

```powershell
# 方式一：直接运行启动器（会开一个持续运行的窗口）
scripts\start-server.cmd
```
启动后日志出现 `服务器已启动`、显示 `HTTP: http://localhost:80` 即成功。
本地验证：`curl http://localhost:80` 应返回 HTML。

### 3.1 让服务开机自启、崩溃自动重启（推荐 NSSM）
下载 [NSSM](https://nssm.cc/) 解压得到 `nssm.exe`，管理员 PowerShell 执行：
```powershell
nssm install EasyAgent "C:\Windows\System32\cmd.exe" "/c D:\easyagent\scripts\start-server.cmd"
nssm set EasyAgent AppDirectory "D:\easyagent"
nssm set EasyAgent DisplayName "EasyAgent Server"
nssm start EasyAgent
```
之后可用 `nssm stop/start/restart EasyAgent` 管理服务，并自动随系统启动。

## 4. 域名与网络（让外网能访问）

> **两条路，先看结论**：
> - ✅ **路径 A（推荐，立即可用，零成本）**：放行 `TCP 3456`，用 `http://82.156.71.231:3456`。非 80/443 端口，**绕开备案拦截**，EasyAgent 已监听 `0.0.0.0:3456`。
> - ⏳ **路径 B（干净域名 `http://CCCN.fable5.icu`）**：需域名已备案、或接入 EdgeOne「不含中国大陆」加速。**未备案域名直连 80 会被腾讯云拦截**，别直接 A 记录指向服务器 IP。
>
> 实测背景与方案对比见 👉 `docs/61_部署讨论与结论复盘.md`。

### 4.1 路径 A：放行 3456 直接访问（推荐先做这个）

服务器是**腾讯云轻量应用服务器**，其「防火墙」= CVM 安全组。在轻量控制台放行即可：

1. 打开 **https://console.cloud.tencent.com/lighthouse/instance/index**
2. 点实例 `82.156.71.231` → 顶部 **「防火墙」** → **「添加规则」**：
   - 应用类型：`自定义`
   - 协议：`TCP`
   - 端口：`3456`
   - 策略：`允许`
   - 来源：`0.0.0.0/0`（仅自己访问可填你 IP/32）
3. **确定**，立即生效（轻量默认已放 80/443/22，无需动 Windows 防火墙）。

验证：浏览器打开 **`http://82.156.71.231:3456`**（务必用 `http://` 而非 `https://`）。
看到"网站不安全"是 http 明文常态提示，**点「高级 → 继续访问」即可**进入 EasyAgent 控制台。

### 4.2 路径 B：用干净域名 CCCN.fable5.icu（需备案或 EdgeOne）

仅在「域名已备案」或「走 EdgeOne 不含大陆加速」时可行：

1. **DNS 解析**：域名服务商处为 `CCCN.fable5.icu` 添加 **A 记录 / CNAME**（CNAME 指向 EdgeOne 给的地址，若已备案则 A 记录 → `82.156.71.231`）。生效检查：`nslookup CCCN.fable5.icu`。
2. **放行 80**：轻量防火墙 +（必要时）Windows 防火墙放行 `TCP 80`：
   ```powershell
   New-NetFirewallRule -DisplayName "EasyAgent HTTP 80" -Direction Inbound -Protocol TCP -LocalPort 80 -Action Allow
   ```
3. **⚠️ 未备案拦截**：若 `fable5.icu` 未备案且直接 A 记录指向服务器 IP，腾讯云会在网络层返回"未完成备案"页（nginx 无日志）。此情形请退回路径 A，或按 `61` 文档走 EdgeOne 付费套餐。

### 4.3 验证外网
- 路径 A：`curl.exe http://82.156.71.231:3456/` 应返回含 `<title>EasyAgent Dashboard</title>` 的页面。
- 路径 B：`http://CCCN.fable5.icu` 应看到 EasyAgent 登录/首页；对话走 `ws://CCCN.fable5.icu/ws`，同源无需额外配置。

### 4.4 CORS 同源放行（静态资源 500 的根因与处理）⚠️ 重点

EasyAgent 用 **`cors` 中间件**控制“哪些源可以读取本 API / 携带凭据”。**这是安全控件，不要删除**——它防止第三方恶意网站在受害者浏览器中读取你的 API 响应。早期曾误把 `cors` 整包移除换成自实现中间件，属于过度改动；正确做法是**保留 `cors`，只修正它的放行逻辑**。

#### 现象
- 根页面 `http://<域名或IP>:3456/` 能打开（200），但页面里的 `<script>`/`<link>`（JS/CSS）返回 **500**，响应头带 `content-security-policy: default-src 'none'` + `x-powered-by: Express` + 1361B HTML（Express 默认错误页）。
- DevTools Network 里该资源请求带 `Origin: http://<域名或IP>:3456`，服务器日志报 `Error: Not allowed by CORS`。
- `curl` 不带 `Origin` 头测全是 200，但浏览器必 500 —— 这就是“自己测正常、用户却 500”的原因。

#### 根因
浏览器加载**同源子资源**（JS/CSS）时**也会带 `Origin` 头**。若该 `Origin` 不在 CORS 白名单，`cors` 会拒绝并抛错 → 静态资源 500。
而根页面 `/` 是顶层导航、**不带 Origin** → 放行 → 200，所以呈现“首页能开、资源 500”。
关键坑：**服务器若在 NAT 后**（本机网卡是私网 IP，对外公网 IP 在路由器上、不绑定本机网卡），不能靠“网卡 IP”判断同源。

#### 当前实现（已修复，保留 cors 包）
`packages/server/src/index.ts`：
1. 前置一个**同源预判定中间件**（它有权读 `req.headers.host`）：比较 `Origin` 的 `host:port` 与请求 `Host` 头是否一致；
   - 一致（同源，含公网 IP / 域名 / 局域网 IP，NAT 透明生效）→ 摘除 `Origin`，`cors` 按“无 origin（同源导航）”放行；
   - 不一致 → 保留 `Origin`，交给 `cors` 按白名单处理。
2. `cors` 放行策略：`无 Origin（同源导航）` / `file://（Electron Desktop）` / `localhost / 127.0.0.1（本地开发）` / `CORS_ORIGIN` 环境变量白名单（独立前端域名）。
3. **安全性不变**：第三方恶意站点（Origin 与 Host 不符且不在白名单）仍被 `cors` 拒绝。

#### 以后用域名 / 独立前端怎么办
- **同域部署**（前端与 API 同一主机，如 `cccn.fable5.icu` 反代到 3456）：**无需任何配置**，同源预判定自动放行。
- **独立前端域名**（如前端 `app.example.com`、API `api.example.com`）：在**服务器启动环境**设置 `CORS_ORIGIN` 为前端源（多个逗号分隔），`cors` 即把该源加入白名单。例如写进 `start.bat`：
  ```powershell
  $env:CORS_ORIGIN="https://app.example.com,https://app.example.com:3456"
  ```

#### 排查命令
```powershell
# 模拟浏览器“带 Origin”请求资源，应返回 200 且响应头含 Access-Control-Allow-Origin
curl.exe -s -o NUL -w "HTTP %{http_code}\n" `
  -H "Origin: http://82.156.71.231:3456" `
  http://82.156.71.231:3456/assets/index-xxxx.js
# 若仍是 500，说明 server 未部署“同源预判定”版本，需重新构建并部署
```

## 5. 升级到 HTTPS（后续，可选）

将来要上证书时，推荐用 **Caddy**（单文件、自动申请 Let's Encrypt）：

1. 把 `start-server.cmd` 的 `PORT` 改为 `3456`（避开 80）。
2. 安装 Caddy，新建 `Caddyfile`：
```
CCCN.fable5.icu {
    reverse_proxy localhost:3456
}
```
3. 运行 `caddy run`（或注册为服务）。Caddy 会自动申请/续期证书，
前端 `ws` 自动升级为 `wss://CCCN.fable5.icu/ws`，无需改代码。

> ⚠️ **Caddy 域名方案的前提**：上述 `CCCN.fable5.icu` + Caddy 自动签证书，**仅在域名已备案、或经 EdgeOne「不含中国大陆」加速时有效**。
> 实测 `fable5.icu` 未备案，访问 `https://CCCN.fable5.icu` 会在 TLS 握手阶段被腾讯云 SNI 拦截，证书再真也到不了 nginx。
> 纯 IP（`https://82.156.71.231:8443`）+ 自签证书 + 本机信任根证书是零成本 HTTPS 替代路径（IP 直连不触发 SNI）。
> 详见 `docs/61_部署讨论与结论复盘.md` §三/§四。

## 6. 常见问题排查

| 现象 | 原因 | 解决 |
|------|------|------|
| `pnpm install` 报 better-sqlite3 编译失败 | 缺 VS Build Tools / Python | 重做 1.4，确认 `cl` 与 `python` 可用 |
| 启动报 `MODULE_VERSION` / `.node` 加载错误 | better-sqlite3 与运行 Node ABI 不符 | 重跑 `node scripts/sqlite3-loader.mjs system`；确认 Node 不是 24 |
| 启动报 `preinstall` 拦截 Node 24 | 服务器装了 Node 24 | 降到 20/22 LTS |
| 浏览器打不开，但 `localhost` 正常 | 防火墙/安全组未放行 80，或 DNS 未生效 | 检查 4.1/4.2 |
| 端口 80 被占用（IIS/Skype 等） | 80 被别的服务占用 | 停掉占用程序，或改用 3456 + Caddy 反向代理 |
| 页面能开但对话一直“正在思考” | WebSocket 未连通 | 确认同源 `ws://域名/ws`；检查安全组是否放行（WS 复用 80） |
| 首页能开，但 JS/CSS 子资源 500（`CSP: default-src 'none'` + Express 错误页） | CORS 中间件拒绝同源子资源的 `Origin`（NAT/公网 IP 不在白名单） | 确认已部署含“同源预判定”的 server（见 §4.4）；独立前端域名需设 `CORS_ORIGIN` |
