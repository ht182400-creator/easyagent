# EasyAgent 服务器部署指南（Windows 云服务器）

> 目标：把 EasyAgent 部署到 Windows 云服务器 `82.156.71.231`，通过域名 `http://CCCN.fable5.icu` 访问。
> 先跑通 HTTP(80)，SSL 证书后续再加（见文末升级方案）。
> 仓库：`https://github.com/ht182400-creator/easyagent.git`（公开，HTTPS 可直接 clone）

---

## 0. 部署架构（无需改动任何业务代码）

```
浏览器 → CCCN.fable5.icu (DNS A 记录 → 82.156.71.231:80)
              │  同源访问（不会触发 CORS 预检）
              ▼
   EasyAgent Server (Node, HOST=0.0.0.0, PORT=80)
     ├─ /         → 托管 packages/web/dist（静态页面）
     ├─ /api/*    → REST API
     └─ /ws       → WebSocket（流式对话）
```

**为什么这么简单**：Server 本身用 `express.static` 同时托管前端页面 + API + WebSocket；
前端默认 `apiBase:''`、`wsBase:'/ws'`（相对路径），同源部署时自动变为 `CCCN.fable5.icu/api`
与 `ws://CCCN.fable5.icu/ws`。所以**只需在服务器上跑一个 Node 进程**即可。

> 唯一坑：better-sqlite3 是原生模块，必须与服务器 Node 的 ABI 完全一致。
> 项目 `preinstall` 会**拦截 Node ≥ 24**，且 `pnpm install` 会自动编译该模块。
> 因此服务器必须用 **Node 18 / 20 / 22 LTS**（推荐 20 或 22 LTS）。

## 1. 前置环境准备（服务器上，仅需一次）

通过远程桌面（RDP）登录 `82.156.71.231`，用 **管理员 PowerShell** 执行：

### 1.1 安装 Git
```powershell
winget install Git.Git
```

### 1.2 安装 Node.js LTS（推荐 22 LTS，严禁 24）
```powershell
winget install OpenJS.NodeJS.LTS
# 安装后重新打开 PowerShell，确认版本在 18~22 之间
node -v
```
> 若已误装 Node 24：用 `winget uninstall OpenJS.NodeJS` 卸载，再装 LTS。

### 1.3 安装 pnpm
```powershell
npm install -g pnpm@11.7.0
pnpm -v
```

### 1.4 安装 C++ 编译工具链（better-sqlite3 必须）
better-sqlite3 在 Windows 上需从源码编译，依赖：
- **Visual Studio 2022 Build Tools** + “使用 C++ 的桌面开发” 工作负载
- **Python 3**（node-gyp 需要）

```powershell
# C++ 生成工具
winget install Microsoft.VisualStudio.2022.BuildTools --override "--add Microsoft.VisualStudio.Workload.VCTools;includeRecommended"
# Python
winget install Python.Python.3.12
```
安装后**重启 PowerShell**，确认：
```powershell
python --version
# 以及 cl 可用（VS 开发者命令提示符中）
```

## 2. 拉取代码并部署

### 2.0 最简方式（推荐）：一行命令全自动
在服务器**管理员 PowerShell** 里粘贴一行，脚本会自动装环境（Git/Node LTS/pnpm/VS 构建工具/Python）
→ 克隆 → 部署 → 启动：
```powershell
irm https://raw.githubusercontent.com/ht182400-creator/easyagent/main/scripts/bootstrap-server.ps1 | iex
```
> VS Build Tools 装完可能需重启；重启后再次粘贴同一行即可（已装项会自动跳过）。
> 脚本源码见 `scripts/bootstrap-server.ps1`，可加 `-CloneDir D:\other` 改路径。

### 2.1 手动方式：克隆仓库
```powershell
# 建议放到 D:\easyagent 或 C:\easyagent
cd D:\
git clone https://github.com/ht182400-creator/easyagent.git easyagent
cd easyagent
```

### 2.2 一键部署（校验环境 + 装依赖 + 编译 + 构建）
```powershell
pwsh scripts/deploy-server.ps1
```
脚本会自动：校验 Node 版本 → `git pull` → `pnpm install`（编译 better-sqlite3）→
激活系统 Node 版原生模块 → 构建 `core / langgraph / server / web`。

若想构建后直接启动，加 `-Start`：
```powershell
pwsh scripts/deploy-server.ps1 -Start
```

## 3. 启动服务

> ⚠️ **绑定 80 端口需要管理员权限**。请用“管理员 PowerShell / 右键以管理员身份运行”。

```powershell
# 方式一：直接运行启动器（会开一个持续运行的窗口）
scripts\start-server.cmd
```
启动后日志出现 `服务器已启动`、显示 `HTTP: http://localhost:80` 即成功。
本地验证：`curl http://localhost:80` 应返回 HTML。

### 3.1 让服务开机自启、崩溃自动重启（推荐 NSSM）
下载 [NSSM](https://nssm.cc/) 解压得到 `nssm.exe`，管理员 PowerShell 执行：
```powershell
nssm install EasyAgent "C:\Windows\System32\cmd.exe" "/c D:\easyagent\scripts\start-server.cmd"
nssm set EasyAgent AppDirectory "D:\easyagent"
nssm set EasyAgent DisplayName "EasyAgent Server"
nssm start EasyAgent
```
之后可用 `nssm stop/start/restart EasyAgent` 管理服务，并自动随系统启动。

## 4. 域名与网络（让外网能访问）

### 4.1 DNS 解析
在域名服务商处为 `CCCN.fable5.icu` 添加 **A 记录** → `82.156.71.231`。
等待生效：`nslookup CCCN.fable5.icu`（返回 82.156.71.231 即 OK）。

### 4.2 放行端口（两道关卡都要开）
1. **云厂商安全组**（腾讯云控制台）：入站规则放行 **TCP 80**（源 `0.0.0.0/0`）。
2. **Windows 防火墙**：管理员 PowerShell：
```powershell
New-NetFirewallRule -DisplayName "EasyAgent HTTP 80" -Direction Inbound -Protocol TCP -LocalPort 80 -Action Allow
```

### 4.3 验证外网
浏览器打开 `http://CCCN.fable5.icu`，应看到 EasyAgent 登录/首页。
对话功能走 `ws://CCCN.fable5.icu/ws`，同源无需额外配置。

## 5. 升级到 HTTPS（后续，可选）

将来要上证书时，推荐用 **Caddy**（单文件、自动申请 Let's Encrypt）：

1. 把 `start-server.cmd` 的 `PORT` 改为 `3456`（避开 80）。
2. 安装 Caddy，新建 `Caddyfile`：
```
CCCN.fable5.icu {
    reverse_proxy localhost:3456
}
```
3. 运行 `caddy run`（或注册为服务）。Caddy 会自动申请/续期证书，
前端 `ws` 自动升级为 `wss://CCCN.fable5.icu/ws`，无需改代码。

## 6. 常见问题排查

| 现象 | 原因 | 解决 |
|------|------|------|
| `pnpm install` 报 better-sqlite3 编译失败 | 缺 VS Build Tools / Python | 重做 1.4，确认 `cl` 与 `python` 可用 |
| 启动报 `MODULE_VERSION` / `.node` 加载错误 | better-sqlite3 与运行 Node ABI 不符 | 重跑 `node scripts/sqlite3-loader.mjs system`；确认 Node 不是 24 |
| 启动报 `preinstall` 拦截 Node 24 | 服务器装了 Node 24 | 降到 20/22 LTS |
| 浏览器打不开，但 `localhost` 正常 | 防火墙/安全组未放行 80，或 DNS 未生效 | 检查 4.1/4.2 |
| 端口 80 被占用（IIS/Skype 等） | 80 被别的服务占用 | 停掉占用程序，或改用 3456 + Caddy 反向代理 |
| 页面能开但对话一直“正在思考” | WebSocket 未连通 | 确认同源 `ws://域名/ws`；检查安全组是否放行（WS 复用 80） |
