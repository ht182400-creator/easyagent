## 二、端口分配原则

1. **345x 段**：EasyAgent 后端服务（3456 主后端 + 文档浏览器静态托管，3455 LangGraph Demo）
2. **417x-518x 段**：前端开发/预览服务器（4173 预览，5173 Web Dev，5183 Desktop Dev，5184 Doc_project）
3. **889x 段**：管线/发布相关（8898 仪表板，8899 同步脚本）
4. **114xx 段**：外部服务（Ollama 11434）
5. **3000 段**：用户项目预览（PreviewTool 自动选择 3000-3999）

## 十、方案 D：插件市场 CI/CD 自动构建架构

> 新增于 2026-07-02，用于解决第三方插件预构建产物（`dist/`）的分发问题。

### 10.1 问题背景

EasyAgent 第三方插件本质上是一个 **Vite React 前端项目**，需要构建生成 `dist/` 静态文件才能在 iframe 中运行。

| 原方案 | 描述 | 问题 |
|--------|------|------|
| 插件目录不含 dist | GitHub 只上传源码，用户下载后没有 dist/ | 文档浏览器无法运行 |
| `D:\Work_Area\AI\Doc_project` | Doc_project 是独立项目，不在插件目录内 | 架构混乱，路径硬编码 |

### 10.2 方案 D：GitHub Actions 自动构建 + Release Asset

**核心思路**：插件开发者在 GitHub 推送版本 tag → GitHub Actions 自动构建 `dist/` → 上传 `dist.zip` 到 Release Assets → EasyAgent 安装时下载 `dist.zip` 而非源码。

```
插件开发者推送 v1.0.0
        ↓
GitHub Actions (build.yml)
  ├─ checkout
  ├─ pnpm install
  ├─ pnpm build      → 生成 dist/
  ├─ zip -r dist.zip dist/
  └─ 上传到 Release Assets (dist.zip)
        ↓
EasyAgent 安装插件
  ├─ 获取 Release 信息
  ├─ 检查 assets 中是否有 dist.zip
  │     ├─ 有 → 下载 dist.zip → 解压到插件目录
  │       → dist/ 在 ~/.easyagent/plugins/obsidian-doc-viewer/dist/
  │       → Server 动态托管 ~/.easyagent/plugins/obsidian-doc-viewer/dist/
  └─ 无 → 降级下载源码 zip（无 dist 的旧插件兼容）
```

### 10.3 涉及的文件改动

| 文件 | 改动内容 |
|------|----------|
| `PluginMarketService.ts` | 安装时优先下载 Release Asset `dist.zip`；新增 `downloadAsset()` 方法 |
| `server/src/index.ts` | `docViewerDistPath` 从硬编码改为动态查找 `~/.easyagent/plugins/<plugin>/dist/` |
| **GitHub 仓库需新增** `.github/workflows/build.yml` | GitHub Actions workflow（见下方） |
| **插件 `vite.config.ts`** | 构建输出到 `dist/`，base 为 `/`（托管时由 Server 设置路径前缀） |

### 10.4 GitHub Actions Workflow（需在插件仓库创建）

在插件 GitHub 仓库根目录创建 `.github/workflows/build.yml`：

```yaml
name: Build & Release

on:
  push:
    tags:
      - 'v*'

jobs:
  build:
    runs-on: ubuntu-latest
    permissions:
      contents: write

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install dependencies
        run: npm ci

      - name: Build
        run: npm run build

      - name: Create dist.zip
        run: zip -r dist.zip dist/

      - name: Create GitHub Release
        uses: softprops/action-gh-release@v2
        with:
          files: dist.zip
          generate_release_notes: true
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

> **注意**：项目需有 `package-lock.json`（由 `npm install` 生成）。pnpm 在某些 GitHub Actions 环境中存在兼容性问题，CI 使用 npm 确保稳定性。

**发布流程**：
```bash
# 1. 本地构建验证（npm）
cd easyagent-plugin-obsidian-doc-viewer
npm install
npm run build

# 2. 生成 package-lock.json（CI 依赖此文件）
npm install  # 生成 package-lock.json
git add package-lock.json

# 3. 提交并打版本标签
git add .
git commit -m "feat: v1.0.0 release"
git tag v1.0.0
git push origin main --tags

# 4. GitHub Actions 自动构建并上传 dist.zip 到 Release
#    Release 页面: https://github.com/ht182400-creator/easyagent-plugin-obsidian-doc-viewer/releases
```

### 10.5 插件 vite.config.ts 要求

插件的 `vite.config.ts` 构建配置需满足：

```ts
// vite.config.ts
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';

export default defineConfig({
  plugins: [react()],
  base: '/',  // ⚠️ 必须设为 '/'，Server 通过 /doc-viewer/ 路径托管
  build: {
    outDir: 'dist',
    emptyOutDir: true,
  },
});
```
