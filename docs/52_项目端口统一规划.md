## 二、端口分配原则

1. **345x 段**：EasyAgent 后端服务（3456 主后端，3455 LangGraph Demo）
2. **517x-518x 段**：前端开发服务器（5173 Web，5183 Desktop）
3. **889x 段**：管线/发布相关（8898 仪表板，8899 同步脚本）
4. **114xx 段**：外部服务（Ollama 11434）
5. **3000 段**：用户项目预览（PreviewTool 自动选择 3000-3999）

## 三、已修复的冲突

### 3.1 LangGraph Demo ↔ 主后端（已修复 ✓）

- **问题**：`packages/langgraph/scripts/demo-server.ts` 硬编码 `PORT = 3456`，与 EasyAgent 主后端端口冲突
- **影响**：同时运行 EasyAgent 主后端和 LangGraph Demo 时端口被占用
- **解决**：LangGraph Demo 改为 **3455**
- **修改日期**：2026-06-29
- **涉及文件**：
  - `packages/langgraph/scripts/demo-server.ts` — PORT 3456 → 3455
  - `packages/langgraph/start-demo.bat` — echo 输出端口号
  - `packages/langgraph/docs/05_启动与执行指南_新手向.md` — 文档端口引用

### 3.2 管线端口不一致（待修复）

- **问题**：`docs/pipeline/server.mjs` 默认 8898，但 `scripts/pipeline-auto-sync.ps1` 和 `release-publish.bat` 使用 8899
- **建议**：统一为 8898（修改同步脚本）或 8899（修改 server.mjs 默认值）

### 3.3 安装脚本端口误导（待修复）

- **问题**：`scripts/install.ps1` 和 `scripts/install.sh` 声称 Web UI 在 `http://localhost:3000`，实际后端监听 3456
- **建议**：修正安装脚本中的端口引用为 3456

## 五、启动方式

| 服务 | 命令 |
|------|------|
| 主后端 API | `start-backend.bat` → `http://localhost:3456`（可通过 `set API_PORT=xxxx` 覆盖） |
| Web 前端 | `start-frontend.bat` → `http://localhost:5173`（通过 Vite proxy 连接 API_PORT） |
| LangGraph Demo | `cd packages/langgraph && start-demo.bat --web` → `http://localhost:3455`（可通过 `set LANGGRAPH_PORT=xxxx` 覆盖） |
| 管线仪表板 | `node docs/pipeline/server.mjs 8898` → `http://localhost:8898` |

### 环境变量速查

| 变量名 | 默认值 | 作用范围 |
|--------|--------|----------|
| `PORT` | `3456` | `packages/server` 主后端监听端口 |
| `API_PORT` | `3456` | Electron 主进程、Vite 代理目标、前端 API 基址 |
| `LANGGRAPH_PORT` | `3455` | LangGraph Demo Web 服务端口 |

> **注意**：如果同时修改 `PORT` 和 `API_PORT`，请保持一致指向同一服务。
