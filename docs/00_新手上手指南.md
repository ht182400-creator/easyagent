# EasyAgent 新手上手指南

> **目标读者**：第一次接触 EasyAgent 的开发者  
> **预计阅读 + 操作时间**：30 分钟  
> **更新日期**：2026-06-27  
> **适用版本**：v0.6.6+

---

## 一、这是什么项目？

**EasyAgent** 是一个集成了中国主流大模型的开源 AI 编程助手，有 CLI / Web Dashboard / Desktop 三种使用方式。

| 使用方式          | 入口                           | 适合          |
| ----------------- | ------------------------------ | ------------- |
| **CLI**           | 终端 `easyagent chat`          | 极客、CI 脚本 |
| **Web Dashboard** | 浏览器 `http://localhost:5173` | 轻量使用      |
| **Desktop EXE**   | 安装后双击运行                 | 普通用户      |

技术栈：TypeScript + React 18 + Electron 30 + SQLite + Vitest + tsup + Tailwind CSS

## 二、环境准备（5 分钟）

### 必须安装

```bash
# 确认以下工具已装
git --version     # >= 2.x
node --version    # >= 18.0.0, < 24.0.0
pnpm --version    # >= 8.x
```

### Git 配置

```bash
git config --global core.quotepath false   # 避免中文文件名显示为八进制转义
```

### 克隆仓库

```bash
git clone https://github.com/ht182400-creator/easyagent.git
cd easyagent
pnpm install
```

> ⚠️ `pnpm install` 后 postinstall 脚本会自动编译 better-sqlite3，可能需要 Visual Studio Build Tools 2022。

## 四、理解项目结构（10 分钟阅读）

```
EasyAgent/
├── packages/
│   ├── core/         # 🔧 核心库（Agent、工具、MCP、多模型适配器）
│   ├── langgraph/    # 🆕 LangGraph 工作流引擎 (StateGraph + Checkpoint)
│   ├── server/       # 🌐 Express 后端 (端口 3456, 双引擎切换)
│   ├── desktop/      # 🖥️ Electron 桌面应用
│   ├── web/          # 🌍 Web Dashboard (独立前端)
│   ├── frontend/     # 🎨 共享前端组件 (Desktop + Web 共用)
│   └── cli/          # ⌨️ 命令行工具
├── docs/             # 📖 项目文档（26 篇）
├── scripts/          # 🔧 构建/发布/同步脚本 (.mjs)
├── .codebuddy/
│   └── memory/       # 🧠 AI 开发日志 + 长期记忆
├── .github/workflows/# ⚙️ CI/CD 配置
├── version.json      # 唯一版本号
├── CHANGELOG.md      # 版本更新日志
├── build.bat         # 构建入口
├── release-publish.bat  # 本地全流程发布
└── release-server.bat  # CI/CD 模式发布
```

> 🔴 **关键理解**：修改 `packages/server/src/` 源码后，Desktop tsup 会把 server 代码 **内联进 main.js**。所以必须重编 desktop 才能生效。

## 五、第一次构建 Desktop EXE（15 分钟）

### 5.1 理解构建链

```
修改源码 → tsup 编译 → Vite 打包前端 → electron-builder 打包 EXE
                                                          ↓
                                            EasyAgent-0.6.3-win-x64.exe  (NSIS 安装包)
                                            EasyAgent-0.6.3-win-x64.exe.blockmap
                                            latest.yml  (自动更新元数据)
```

### 5.2 构建命令

```bash
build.bat                     # 快速测试 (~60s), 输出 win-unpacked/
build.bat --release           # 完整 NSIS 安装包 (~3-5min), 输出 .exe
build.bat --verify            # 仅预检查, 不构建
```

### 5.3 构建前必检（来自 `14_构建前必检清单.md`）

- [ ] 关闭 VS Code 或排除 `**/packages/desktop/release/**`
- [ ] 确保无 EasyAgent.exe / electron.exe 进程运行
- [ ] `version.json` 版本号正确
- [ ] 运行过 `pnpm install` 确保依赖完整

### 5.4 输出在哪

```
packages/desktop/release/
├── EasyAgent-0.6.3-win-x64.exe          # NSIS 安装包 (~105 MB)
├── EasyAgent-0.6.3-win-x64.exe.blockmap  # 增量更新块映射 (~200 KB)
└── latest.yml                            # 自动更新元数据 (~200 B)
```

## 六、第一次发布版本（20 分钟）

### 6.1 两种发布方式，选一个

| 方式           | 命令                  | 构建在哪      | 需要什么                     |
| -------------- | --------------------- | ------------- | ---------------------------- |
| **本地全流程** | `release-publish.bat` | 你的电脑      | MSVC + electron-builder 环境 |
| **CI/CD 模式** | `release-server.bat`  | GitHub 服务器 | 无特殊要求，推送标签即可     |

新手推荐用 **CI/CD 模式**（`release-server.bat`），因为不需要本地构建环境。

### 6.2 CI/CD 模式操作步骤

```bash
# 1. 运行服务器端发布脚本
release-server.bat

# 2. 依次选择：
#    - 升级类型：patch（Bug修复）/ minor（新功能）/ major（重大更新）
#    - 确认推送标签

# 3. 脚本自动完成：
#    ✅ 版本号更新 (version.json + 7 个 package.json)
#    ✅ CHANGELOG 更新
#    ✅ git commit + tag + push
#    ✅ GitHub Actions 自动触发构建（约 5-10 分钟）

# 4. 监控 CI 进度
#    打开：https://github.com/ht182400-creator/easyagent/actions

# 5. CI 完成后
#    GitHub Release 自动创建，3 个产物上传完成
```

### 6.3 版本号规则

```
v0.6.6
│ │  └── patch  (Bug 修复): 0.6.5 → 0.6.6
│ └──── minor  (新功能):   0.6.0 → 0.7.0
└────── major  (重大更新):  0.x.x → 1.0.0
```

## 七、关键概念速查

### Desktop vs Web 两个独立前端

EasyAgent 有 **两套独立的前端代码**，不共享：

|          | Web Dashboard               | Desktop 应用                     |
| -------- | --------------------------- | -------------------------------- |
| 代码位置 | `packages/web/src/`         | `packages/desktop/src/renderer/` |
| 路由     | BrowserRouter (`/sessions`) | HashRouter (`/#/sessions`)       |
| HTTP     | 原生 `fetch()`              | `apiFetch()` (已内置 `.json()`)  |
| 环境     | 浏览器                      | Electron (`file://` + IPC)       |

> 🔴 **修改功能时必须两边独立修改**，不要假设代码共享。

### latest.yml ≠ release.yml

新手最易混淆的两个文件：

| 文件                            | 在哪                        | 是什么                    | 自动生成？  |
| ------------------------------- | --------------------------- | ------------------------- | :---------: |
| `latest.yml`                    | `packages/desktop/release/` | electron 自动更新元数据   |     ✅      |
| `.github/workflows/release.yml` | `.github/workflows/`        | GitHub Actions CI/CD 配置 | ❌ 手动维护 |
