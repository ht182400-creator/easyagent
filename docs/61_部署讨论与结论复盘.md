# 61_服务器部署讨论与结论复盘（EasyAgent Web 上线 + cccn.fable5.icu 访问路径）

> 文档类型：部署讨论 / 决策复盘
> 编写日期：2026-07-11
> 涉及对象：腾讯云轻量应用服务器 `82.156.71.231`、域名 `fable5.icu` 体系（`micro.fable5.icu` / `cccn.fable5.icu`）、EdgeOne（Pages / 站点加速）
> 目标产物：EasyAgent Web 版可在公网访问（先 HTTP，后 HTTPS）

---

## 一、背景

### 1.1 业务目标
将 EasyAgent（Node 后端 + WebSocket + React 前端）以 **Web 版** 形态部署到一台 Windows 云服务器，并通过域名 `cccn.fable5.icu` 对外访问。原计划是「先 HTTP(80) 跑通，后续再上 SSL」。

### 1.2 基础设施事实
| 项目 | 事实 |
|---|---|
| 服务器 | `82.156.71.231`，**腾讯云轻量应用服务器**（Lighthouse）。其「防火墙」= CVM 的「安全组」等效物 |
| 操作系统 | Windows（默认 shell 为 PowerShell） |
| 默认已放行端口 | 轻量防火墙默认放行 80 / 443 / 22 |
| 既有 Web 服务 | BT 面板管理的 `nginx`（`C:/www/nginx/nginx.exe`）占用 80 端口，反代 `micro` 应用（micro 跑在 3000） |
| 域名体系 | `fable5.icu` 未在工信部备案；同域名下 `micro.fable5.icu` 已能正常访问 |
| 应用监听 | EasyAgent 监听 `0.0.0.0:3456`（已实测可公网直连，纠正了早期"127.0.0.1"误记） |

### 1.3 关键约束（部署规则）
- `preinstall` 拦截 Node≥24；服务器必须用 **Node 18/20/22 LTS**。
- 启动前须 `node scripts/sqlite3-loader.mjs system` 激活系统 Node 版原生模块（better-sqlite3 ABI）。
- 构建链：`core → langgraph → server → web`，Server 读取 `packages/web/dist`，`express.static` 同源托管页面 + `/api/*` + `/ws`，前端用相对路径无 CORS。

## 二、问题陈述

部署推进过程中，依次暴露出 4 个环环相扣的问题：

1. **域名访问被"备案页"拦截**
   把 `cccn.fable5.icu` 的 DNS A 记录直接指向 `82.156.71.231:80` 后，浏览器打开返回"您的网站未完成备案"页面，nginx 的 access.log 中该域名请求数为 0（说明拦截发生在到达 nginx 之前）。
   - 疑问：`micro.fable5.icu` 同在 `fable5.icu` 域名下，为何它能免备案正常访问？

2. **"取消 Pages 部署腾出 EdgeOne 资源"是误判**
   曾假设 cccn 绑定不上 EdgeOne「站点加速」是因为免费套餐的 1 个站点配额被 `micro`（Pages）占用，于是想删 Pages 站点腾额度。

3. **纯 IP 无法获得浏览器信任的免费 HTTPS 证书**
   用户希望给 `cccn` 去掉浏览器"网站不安全"提示。但 `http://82.156.71.231:3456` 是明文，Let's Encrypt 等公开 CA 不签发 IP 地址证书。

4. **未备案域名 443 被 SNI 拦截， DNS 验证的 Let's Encrypt 方案作废**
   改用"域名 `cccn.fable5.icu` + DNS 验证 Let's Encrypt + nginx 443 反代"方案时，发现即使有真证书，访问 `https://cccn.fable5.icu` 也在 TLS 握手阶段被切断。

## 三、分析

### 3.1 腾讯云备案拦截机制（核心根因）
- **未备案域名 + 大陆 CVM/轻量 公网 IP** 的 80（HTTP）和 443（HTTPS）端口，会在**请求到达服务器之前**被腾讯云网络层拦截：
  - HTTP：返回"未完成备案"页面（应用层完全收不到请求，nginx 无日志）。
  - HTTPS：基于 **SNI（Server Name Indication，明文传输域名）** 检测，对未备案域名在 TLS 握手阶段直接切断。
- **非 80/443 端口（如 3456、8443）不受备案拦截**——这是后续"立即可用方案"的立足点。

### 3.2 为何 micro 能免备案（关键认知）
`micro.fable5.icu` 之所以免备案可用，是因为它走 **EdgeOne 加速，加速区域 = 全球可用区（不含中国大陆）**。腾讯云规定：只要加速节点不含大陆，就无需备案。也就是说，**micro 是经 EdgeOne 边缘回源，并非直连 CVM 公网 IP**。这纠正了"同服务器已工作的域名=直连 CVM"的直觉误解。

### 3.3 EdgeOne 两套产品的本质区别
| 产品 | 用途 | 能否跑 EasyAgent |
|---|---|---|
| **EdgeOne Pages** | 纯静态 / 全栈静态托管 | ❌ 跑不了（无 Node 后端、无 WebSocket） |
| **EdgeOne 站点加速（反向代理）** | 动态反代到源站，支持 `/api/*`、SSR、WebSocket | ✅ 必须用它 |

EasyAgent 是动态应用（Node + WebSocket），**只能用「站点加速」**。

### 3.4 配额体系真相（否定"删 Pages 腾额度"）
通过 EdgeOne API（`DescribePlans` / `DescribeZones`）实测，该账号唯一套餐 `edgeone-3sd1sd2vzs88`（免费）包含**两个彼此独立**的资源：
1. `edgeone-3sd1sd2vzs88`（Type=`pages`、ZoneNumber=1）→ 绑定 `default-pages-zone`（即 micro 的 Pages 站点）。
2. `edgeone-3sd1sf4ozzb9`（Type=`pay-as-you-go` 按量计费、ZoneNumber=**0**）→ 站点加速资源，空闲但**额度=0**。

而 fable5.icu 的站点加速 `zone-3sef5fkv1ymi` 的 `Resources=[]`（空）。
再试 `BindZoneToPlan` 绑按量资源 → **仍报"超过配额上限"**。

**结论：**
- 免费套餐是 **Pages 专用套餐，根本不含「站点加速」站点额度**。
- Pages 资源与站点加速资源是**两套体系**，删 Pages 只会释放 pages 额度，对 cccn 站点加速站点毫无帮助。
- 删 `default-pages-zone` 会**直接让 micro.fable5.icu 下线（不可逆）**，且 cccn 问题一个都解决不了。→ **明确否决"删 Pages"**。

### 3.5 SNI 拦截的对照实证
| 访问目标 | TCP 443 | TLS 握手 | 结果 |
|---|---|---|---|
| `https://cccn.fable5.icu`（带域名 SNI） | 可达（True） | **失败，curl exit 35** | 被腾讯云切断 |
| `https://82.156.71.231:443`（IP 直连，无 SNI） | 可达（True） | **成功**，返回 `HTTP/1.1 200 OK`、`Server: nginx/1.26.2`、`X-Powered-By: Express` 及 micro 页面 | 正常 |

服务器 443 本身就有 nginx 在提供 TLS，IP 直连可握手成功 → 证明 cccn 域名的失败是**腾讯云 SNI 拦截未备案域名**所致。因此"拿真证书就通"是错误预期：**证书再真，握手阶段就被拦，根本到不了 nginx**。

## 四、解决方案对比

| 方案 | 路径 | 成本 | 无警告 HTTPS | 干净域名 | 状态 |
|---|---|---|---|---|---|
| **A. 防火墙放行 3456** | 轻量防火墙加 `TCP:3456` → 用 `http://82.156.71.231:3456` | 0 | ❌（http 明文提示，点"继续访问"即可） | ❌（带 IP+端口） | ✅ **已实施并验证可用** |
| **B. EdgeOne 付费套餐** | 另购真正含「站点加速」额度的套餐 → 绑 `zone-3sef5fkv1ymi` → 加 `cccn.fable5.icu` 加速域名（源站 82.156.71.231:80、开 WS、`/api/*` 与 `/ws` 不缓存）→ 给 CNAME 改 DNS | 花钱 | ✅（自动签受信任证书） | ✅ | 待用户购套餐后实施 |
| **C. 自签证书 + IP 直连** | nginx 用非标准端口（如 `8443`）起 HTTPS + 自签证书反代 3456（含 WS）；IP 直连不触发 SNI 拦截；用户把根证书导入本机"受信任根证书颁发机构" | 0 | ✅（导入后绿锁） | ❌（带 IP+端口） | 备选（未实施） |
| **D. Let's Encrypt + 域名 443** | `cccn.fable5.icu` DNS 验证申请真证书 + nginx 443 反代 | 0 | 预期 ✅ 但 **实际 ✗** | ✅ | ❌ **已否决（SNI 拦截）** |

> 注：方案 D 最初被选中，经实测 443 SNI 拦截后推翻。方案 A 是当前唯一已落地且零成本的可用路径。

## 五、实施过程

### 5.1 远程部署（已落地）
- 用户提供 SSH 凭据（Administrator + 密码），AI 用 `plink.exe` / `pscp.exe` 直连（`-P 22`）。
- 端口冲突：80 被 BT 面板 nginx 占用；采用 **EasyAgent 监听 `PORT=3456 HOST=0.0.0.0`**（早期误记 127.0.0.1，后由公网实测纠正）。
- nginx 新增 `cccn.fable5.icu:80` 反代块 → `127.0.0.1:3456`（含 WebSocket Upgrade 头），原 micro 配置保留，原 `nginx.conf` 备份为 `nginx.conf.bak`。
- 自启动：`ea_server` 计划任务（`/ru SYSTEM /sc onstart /rl highest`，脱离 SSH 会话，断 SSH / 注销 / 重启均存活）运行 `cmd /c C:/easyagent/start.bat`。
- 验证：`curl 127.0.0.1:3456` 与 `curl -H Host:CCCN.fable5.icu 127.0.0.1:80` 均返回 200 + EasyAgent Dashboard。

### 5.2 备案拦截诊断（已落地）
- 确认 `fable5.icu` 未备案 → cccn 直连 IP 的 80 被网络层拦（返回备案页，nginx 无日志）。
- 明确修复方向：要么 cccn 也走 EdgeOne「不含中国大陆」加速（需付费套餐），要么绕开 80 端口。

### 5.3 EdgeOne API 实测（已落地，用于否定错误假设）
- 用 Node 自签名脚本 `temp/eo.mjs`（TC3-HMAC-SHA256）调 `teo` API。
- `DescribeZones` 证明密钥名下只有 1 个 Pages 站点，micro 是 Pages 而非站点加速；`CreateAccelerationDomain` 报 `UnauthorizedOperation`（站点未绑加速资源）；`BindZoneToPlan` 报"超过配额上限"。→ 确立"免费套餐无站点加速额度"结论。
- 已建 `zone-3sef5fkv1ymi`（partial / CNAME / overseas）待用。

### 5.4 防火墙放行 3456（已落地，当前主用方案）
- 用户在轻量控制台 → 实例 → 防火墙 → 添加规则：`TCP:3456`、允许、来源 `0.0.0.0/0`。
- 实测 `curl.exe http://82.156.71.231:3456/` 返回 EasyAgent Dashboard 完整 HTML → **公网已通**。
- `:80` 经 nginx 返回 301（nginx/1.26.2）。

### 5.5 HTTPS 方案验证（已落地，方案 D 被推翻）
- `cccn.fable5.icu` A 记录已指向 82.156.71.231（✓）。
- `https://cccn.fable5.icu` → TCP 443 可达但 TLS 握手失败（exit 35）。
- 对照 `https://82.156.71.231:443`（IP 直连）→ 握手成功、返回 micro 页面 → 确证 SNI 拦截，方案 D 作废。

## 七、踩坑与经验（可复用）

1. **别把"同服务器已工作的域名"等同于"直连 CVM"** —— micro 是经 EdgeOne 而非直连，这点决定了 cccn 也必须走 EdgeOne 才能免备案。
2. **怀疑备案拦截时，先看域名是否走 EdgeOne「不含中国大陆」豁免**；nginx 无访问日志 = 拦截在上游网络层，不是应用故障。
3. **EdgeOne Pages ≠ 站点加速**：动态应用（Node/WebSocket）必须选站点加速，Pages 跑不了。
4. **免费 Pages 套餐不含站点加速额度**，删 Pages 腾不出加速配额，反而会让 micro 下线 —— 不可为。
5. **未备案域名的 443 也会因 SNI 被拦**，"拿真证书就通"是错误预期；IP 直连 443 不受 SNI 影响（实测成功）。
6. **非 80/443 端口（3456 等）绕开备案拦截**，是零成本立即可用的兜底路径。
7. **轻量应用服务器**的「防火墙」即 CVM「安全组」；默认已放 80/443/22，新增端口在防火墙加规则即可，立即生效。
8. **远程 Windows 操作**：用 `plink`/`pscp`；远程命令用 `;` 分隔（PowerShell），不用 cmd 的 `&`；pscp 远程路径用正斜杠。
9. **CORS 整包移除是过度改动，根因只是配置错误**：`cors` 中间件是安全控件（防第三方站读 API），不应删除。静态资源 500 的真因是“同源子资源带 `Origin` 头、但白名单只有 localhost/file://，公网 IP 被拒”。修复应**保留 `cors` 包**，仅在前置一个同源预判定中间件（比较 `Origin` 与 `Host` 是否一致，一致则摘除 `Origin` 让 cors 按同源放行），跨域前端走 `CORS_ORIGIN` 环境变量白名单。NAT 下不能靠 `os.networkInterfaces()` 判断同源（取不到公网 IP）。详见部署指南 §4.4。

## 八、当前状态与后续建议

### 当前可用
- ✅ **`http://82.156.71.231:3456`** 已公网通，EasyAgent 可正常使用（浏览器"网站不安全"提示为 http 明文常态，点"继续访问"即可，非故障）。

### 后续可选（按优先级）
1. **维持现状（A 方案）**：零成本，满足功能访问，仅地址带端口、有 http 提示。
2. **想要干净域名 + 无警告 HTTPS（B 方案）**：另购 EdgeOne 真正含「站点加速」额度的付费套餐（个人版/基础版）。购后由 AI 执行：绑 `zone-3sef5fkv1ymi` → 加 `cccn.fable5.icu` 加速域名（源站 82.156.71.231:80、开 WebSocket、`/api/*` 与 `/ws` 不缓存）→ 给 CNAME 改 DNS。
3. **零成本想要 HTTPS（C 方案）**：nginx 用 `8443` 起自签证书 HTTPS 反代 3456（含 WS），用户导入根证书到本机信任库，访问 `https://82.156.71.231:8443` 绿锁无警告（仍带 IP+端口）。

### 安全提醒
- 本会话使用的腾讯云 API 密钥为**主账号级**，建议用完后到控制台删除。脚本 `temp/eo.mjs` 与密钥均不入库、不写 memory。

### 术语表
- **SNI**：TLS 握手中的明文域名扩展，腾讯云据此识别并拦截未备案域名。
- **EdgeOne Pages**：纯静态托管，无后端。
- **EdgeOne 站点加速**：反向代理，支持动态与 WebSocket，需付费套餐才含站点加速额度。
- **轻量防火墙 / CVM 安全组**：同一概念在不同产品下的名称。
- **partial / CNAME 接入**：EdgeOne 站点接入方式，域名 NS 不迁移，仅改 CNAME。
