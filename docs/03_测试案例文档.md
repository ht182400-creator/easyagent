## 1.5 Server 集成测试 (新增)

### 测试模块：API 端点 (27 用例)
- **/api/health**: 返回200 + status/version/uptime/memory/timestamp
- **/api/status**: 返回model/tokenUsage/sessionCount/toolCount/providerCount
- **/api/version**: 返回版本号 + codename + 发布日期 + changelog
- **/api/version/check**: 检查 GitHub Release 是否有新版本
- **/api/tools**: 工具数组70项 + 分组校验(≥5分组) + 字段完整性
- **/api/plugins**: 插件列表 + 字段校验
- **/api/skills**: 技能列表 + 字段校验
- **/api/config**: GET/PUT 配置读写
- **/api/providers**: 提供商列表(含动态模型) + 连接测试
- **/api/sessions**: 会话列表 + 字段校验
- **/api/im/status**: IM状态返回
- **/api/sandbox/status**: 沙箱状态(docker/sandbox字段)
- **/api/semantic/overview**: 语义概览
- **API 404路由**: `/api/nonexistent` 返回JSON 404
- 🆕 **知识库 API — 双作用域 (9 用例)**:
  - `GET /api/knowledge?scope=project` → 项目知识库数据 + scope 字段
  - `GET /api/knowledge?scope=global` → 全局知识库数据 + scope 字段
  - `GET /api/knowledge/stats/summary` → 合并模式 (scope:merged + project + global)
  - `GET /api/knowledge/stats/summary?scope=global` → global 单域统计
  - `GET /api/knowledge/search?q=test&scope=project` → 搜索 + scope 参数
  - `GET /api/knowledge/search` → 缺少 q 参数返回 400
  - `POST /api/knowledge/import` → 缺少 filePath 返回 400
  - `POST /api/knowledge/import` → 不存在的文件返回 400 (含"文件不存在")
  - `POST /api/knowledge/upload` → 缺少文件返回 400

> 🐛 **修复记录**: 知识点 API 原返回 404，根因为 ① 旧编译产物 `src/index.js` 被 vitest 优先加载 ② 路由顺序 `:id` 在 `stats/summary` 前导致 `stats` 被 :id 捕获。修复: 删除旧 artifact + 调整路由顺序。

### 测试模块：静态文件服务 (6 用例)
- index.html 返回200 + Content-Type text/html
- JS资源返回200 + application/javascript
- CSS资源返回200 + text/css

### 测试模块：WebSocket 服务 (5 用例)

- subscribe协议被接受

## 六、MCP测试在Windows下的特殊处理

MCP客户端测试在Windows环境下存在 `cmd.exe` 对 `node -e` 内联脚本的转义问题，已在所有MCP测试中采用以下方案：

1. **使用临时`.js`脚本文件**：通过 `createMcpScript()` 辅助函数将脚本写入临时文件，避免 `cmd.exe` 转义
2. **MCPClient进程退出清理**：在 `process.on('close')` 事件中 reject 所有 pending requests
3. **脚本默认响应**：所有MCP模拟脚本添加 `else` 分支处理 `shutdown` 等未预期请求
4. **测试后清理**：通过 `cleanMcpDir()` 清理临时脚本目录

### 9.4 测试架构决策

- **WeChatAdapter 私有方法隔离**: esbuild 将 TS `private` 关键字严格编译，`wxDecrypt` / `parseWeChatXML` 无法从外部访问。解决方案: 将加解密算法与 XML 解析提取到独立 `wechat-crypto.test.ts` 测试，WeChatAdapter 仅测公共 API 边界。
- **KnowledgeService 同名文档行为**: 同名 update 更新索引元信息但保留旧 docId(内容块不更新)，已在测试注释中记录。
- **XML 解析限制**: `parseWeChatXML` 正则在嵌套 `<xml>` 中仅匹配外层标签，在 `wechat-crypto.test.ts` 测试中采用平级标签验证。
- **Express 路由顺序**: `:id` 参数路由必须放在所有具体路径路由之后，防止参数值捕获路径段。已在 `stats/summary` 之后注册。
- **全局知识库单例**: `KnowledgeService.getGlobal()` 返回跨进程单例（存储于 `~/.easyagent/knowledge/`），多项目共享。
