# EasyAgent 构建前必检清单 ⚡

> **理念**: 每次构建前 30 秒过一遍，避免 2 小时排错。
> **铁律**: 永远用 `build.bat --release`，禁止手动一步步跑。

---

## 一、快速检查卡（每次必看）

| # | 检查项 | 命令 / 方法 | ❌ 失败时症状 |
|---|--------|------------|------------|
| 1 | **已杀旧进程** | `taskkill /f /im EasyAgent.exe` | electron-builder Access Denied |
| 2 | **版本已同步** | `node scripts/sync-version.mjs` | 版本号不对、electron-updater 检测错误 |
| 3 | **preload 有 CJS 配置** | `type packages\desktop\tsup.preload.config.ts` | `window.easyAgent` undefined、所有 IPC 断 |
| 4 | **dist/renderer 已清** | `rmdir /s/q packages\desktop\dist\renderer` | Vite 缓存旧代码、改的前端不进 EXE |
| 5 | **server dist 已重建** | `rmdir /s/q packages\server\dist && cd packages\server && npx tsup --clean` | CORS 还是旧代码 |

## 二、标准构建流程（唯一正解）

```batch
cd /d "D:\Work_Area\AI\Claude Code  CN"

REM === Step 1: 版本同步 ===
node scripts/sync-version.mjs

REM === Step 2: 构建 ===
build.bat --release
```

**`build.bat --release` 自动完成**:
```
Phase 0  → 杀进程 + 删 release + 清 dist/renderer
Phase 1  → verify-build.cjs (17 类 30+ 项检查)
Phase 2  → core tsup → server tsup 
            → desktop tsup + tsup.preload(CJS) + vite build
Phase 2.5 → sqlite3 切换 Electron 版本
Phase 3  → electron-builder --win (含 Defender 重试)
Phase 3.5 → sqlite3 恢复 System 版本
Phase 4  → 输出验证 (EXE 大小 + asar 内容检查)
```

**产物**:
- 便携版: `packages\desktop\release\win-unpacked\EasyAgent.exe` (~168MB)
- 安装包: `packages\desktop\release\EasyAgent-{version}-win-x64.exe` (~105MB)

## 四、高频坑位速查表

### 🔴 致命级（出问题必排查）

| 坑 | 症状 | 原因 | 预防 |
|----|------|------|------|
| **preload ESM** | `窗空无法连接`、`ea.xxx不可用` | preload 被编译为 ESM 而非 CJS | `tsup.preload.config.ts` 存在且 format=cjs |
| **Vite 缓存** | 改了 Settings.tsx 但 EXE 里是旧 UI | 没清 dist/renderer | build.bat Phase 0 自动清 |
| **tsup 缓存** | 改了 server 代码不生效 | desktop main.js 内联了 server 旧代码 | 先清 server dist 再重编 desktop |
| **desktop 不重编** | 改 server 后 desktop 无变化 | tsup noExternal 内联机制 | 改 server/core 后必须重编 desktop tsup |
| **sqlite3 ABI** | 后端启动失败、cannot connect | MODULE_VERSION 不匹配 | build.bat Phase 2.5/3.5 自动切换 |
| **electron-builder v24** | EXE 仅 0.3MB | pnpm 解析了 v24 alpha | package.json 锁定 "23.6.0" 精确版 |
| **Defender 拦截** | NSIS Can't open output file | Defender 锁定 msi/exe | 5s 重试机制或添加排除项 |

### 🟡 警告级

| 坑 | 症状 | 预防 |
|----|------|------|
| **--dir vs --win** | 找不到 EXE 安装包 | `--release` 模式用 `--win`（默认） |
| **GitHub make_latest** | Release 不是 Latest | 传字符串 `"true"` 不是布尔 `true` |
| **release body 中文** | GitHub API 返回 422 | _release.mjs body 用纯英文 |
| **版本不同步** | 部分 package.json 版本是旧的 | 改完 version.json 后跑 `sync-version.mjs` |

## 五、构建后验证

```batch
REM 1. 检查文件
dir packages\desktop\release\EasyAgent-*.exe

REM 2. 检查 latest.yml 版本号
type packages\desktop\release\latest.yml

REM 3. 本地安装测试
start packages\desktop\release\EasyAgent-0.5.xx-win-x64.exe

REM 4. 检查 GitHub Release
start https://github.com/ht182400-creator/easyagent/releases
```

### 关键检查点

- [ ] EXE 大小 ≈ 105MB（太小说明构建失败）
- [ ] `latest.yml` 中 version 字段正确
- [ ] Settings 页面当前版本号正确
- [ ] `window.easyAgent` 在 DevTools Console 可访问
- [ ] "检查更新"按钮可触发更新检测
- [ ] GitHub Release 有 3 个 assets（EXE + blockmap + latest.yml）
- [ ] Release 被标记为 Latest

## 六、禁忌操作 🚫

| ❌ 禁止 | ✅ 应该 |
|---------|--------|
| 手动 `npx tsup` 一步步跑 | `build.bat --release` |
| `npx electron-builder --dir`（测试时）| `build.bat --fast` |
| 跳过 verify-build.cjs | 让 build.bat 自动执行 |
| 用 `npx` 而非 `pnpm exec` | build.bat 内部使用 pnpm exec |
| 在中文目录/路径中构建 | 纯 ASCII 路径 |
| 改完代码不重编 desktop tsup | 记住 tsup 内联机制 |
