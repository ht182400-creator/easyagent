## 一、先澄清：GitHub Releases 完全够用

### GitHub Releases 的限制

| 指标 | 限制 | EasyAgent 实际情况 |
|------|------|:---:|
| 单文件最大 | **2 GB** | 87 MB ✅ |
| 总 Release 大小 | 无硬性限制 | — |
| 带宽 | 无限制 | — |
| 免费 | 公开仓库完全免费 | ✅ |

### 业界 Electron 应用都在用

| 应用 | 安装包大小 | 分发方式 |
|------|:---:|------|
| VS Code | ~90 MB | GitHub Releases + 自建 CDN |
| Discord | ~80 MB | GitHub Releases |
| Figma | ~85 MB | GitHub Releases |
| Postman | ~120 MB | GitHub Releases |
| Obsidian | ~80 MB | GitHub Releases |
| **EasyAgent** | **87 MB** | **GitHub Releases（推荐）** |

> **结论**：87MB 对 GitHub Releases 来说根本不是问题。如果你没有特殊原因（如国内用户下载 GitHub 慢），直接用它就是最成熟、最标准的方案。

## 三、方案详解

### 方案 A：GitHub Releases（最简单，推荐首选）

**这就是你现在的方案，不需要改任何东西。**

```
GitHub Actions 构建 EXE
  → 自动上传到 GitHub Releases
  → electron-updater 直接读 GitHub API 获取更新
```

**优势**：
- 零额外服务依赖
- electron-updater 原生支持 `provider: "github"`
- 配置已经在 `package.json` 的 `build.publish` 中

**劣势**：
- 国内用户下载可能慢（GitHub 在中国没 CDN）
- 如果仓库被墙则无法更新

**适用**：用户主要在海外，或用户量不大时完全够用。

---

### 方案 B：GitHub Releases + jsDelivr CDN（免费加速）

jsDelivr 可以免费代理 GitHub Releases 文件，国内有 CDN 节点。

```
GitHub Actions 构建 EXE
  → 上传到 GitHub Releases
  → jsDelivr 自动缓存
     URL: https://cdn.jsdelivr.net/gh/ht182400-creator/easyagent@v0.3.1/packages/desktop/release/EasyAgent-0.3.1-win-x64.exe
  → electron-updater 配置 custom provider 指向 jsDelivr
```

**优势**：
- 完全免费
- 国内速度大幅提升（有中国 CDN 节点）
- 无需额外账号

**劣势**：
- jsDelivr 有 50MB 单文件限制（exe 87MB 超了）
- 对大型二进制文件支持不稳定

> ⚠️ **jsDelivr 不适合 87MB 的 EXE**，单文件限制 50MB。

---

### 方案 C：Cloudflare R2（推荐的最佳免费方案）

Cloudflare R2 是 S3 兼容的对象存储，**免出口流量费**，全球 CDN 加速。

```
GitHub Actions 构建 EXE
  → 通过 rclone/aws-cli 上传到 Cloudflare R2
  → 通过 Cloudflare CDN 分发
  → electron-updater 配置 generic provider
```

#### 配置步骤

**1. 注册 Cloudflare 账号**（免费）
```
https://dash.cloudflare.com/sign-up
```

**2. 创建 R2 存储桶**
```
进入 R2 → Create bucket
名称: easyagent-releases
位置: Automatic（或亚太地区）
```

**3. 获取 API Token**
```
R2 → Manage R2 API Tokens → Create API Token
权限: Object Read & Write
记下: Access Key ID + Secret Access Key
```

**4. 设置公开访问**
```
R2 → easyagent-releases → Settings → Public Access
  → 开启 r2.dev 子域（或绑定自定义域名）
  → 公开 URL: https://pub-xxx.r2.dev
```

**5. 创建 GitHub Actions Workflow**

```yaml
# .github/workflows/release-r2.yml
name: Build and Release to R2

on:
  push:
    tags: ['v*']

jobs:
  build-and-upload:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with: { node-version: '20' }

      - uses: pnpm/action-setup@v3
        with: { version: '8' }

      - run: pnpm install --frozen-lockfile

      # 同步版本
      - name: Sync version
        run: |
          $v = "${{ github.ref_name }}" -replace '^v',''
          $j = Get-Content version.json -Raw | ConvertFrom-Json
          $j.version = $v
          $j | ConvertTo-Json -Depth 10 | Set-Content version.json
          node scripts/sync-version.mjs

      # 构建
      - run: cd packages/core && pnpm exec tsup
      - run: cd packages/server && pnpm exec tsup
      - run: cd packages/desktop && pnpm exec tsup && pnpm exec vite build

      # 打包
      - name: Build EXE
        run: cd packages/desktop && pnpm exec electron-builder --win --x64

      # 上传到 R2
      - name: Upload to R2
        uses: ryand56/r2-upload-action@v1
        with:
          account-id: ${{ secrets.R2_ACCOUNT_ID }}
          access-key-id: ${{ secrets.R2_ACCESS_KEY_ID }}
          secret-access-key: ${{ secrets.R2_SECRET_ACCESS_KEY }}
          bucket: easyagent-releases
          source: packages/desktop/release
          destination: /releases/${{ github.ref_name }}/

      # 生成 latest.yml 指向 R2
      - name: Generate update manifest
        run: |
          $version = "${{ github.ref_name }}" -replace '^v',''
          $url = "https://pub-xxx.r2.dev/releases/v$version"
          @"
          version: $version
          files:
            - url: $url/EasyAgent-$version-win-x64.exe
              sha512: $(certutil -hashfile packages/desktop/release/EasyAgent-$version-win-x64.exe SHA512 | findstr /v "hash")
              size: $((Get-Item packages/desktop/release/EasyAgent-$version-win-x64.exe).length)
          "@ | Out-File -Encoding utf8 latest.yml
          
      - name: Upload manifest to R2
        uses: ryand56/r2-upload-action@v1
        with:
          account-id: ${{ secrets.R2_ACCOUNT_ID }}
          access-key-id: ${{ secrets.R2_ACCESS_KEY_ID }}
          secret-access-key: ${{ secrets.R2_SECRET_ACCESS_KEY }}
          bucket: easyagent-releases
          source: latest.yml
          destination: /
```

**6. 修改 electron-builder 配置**

更新 `packages/desktop/package.json` 的 `build.publish`：

```json
{
  "build": {
    "publish": {
      "provider": "generic",
      "url": "https://pub-xxx.r2.dev/",
      "channel": "latest"
    }
  }
}
```

**7. 设置 GitHub Secrets**

```
Settings → Secrets and variables → Actions → New repository secret:

R2_ACCOUNT_ID       = Cloudflare 账户 ID（在 R2 页面右侧）
R2_ACCESS_KEY_ID    = R2 API Token 的 Access Key ID
R2_SECRET_ACCESS_KEY = R2 API Token 的 Secret Access Key
```

**费用**：R2 免费额度 10GB 存储 / 每月，EasyAgent 一个版本约 87MB，等于可存 110+ 个版本。出口流量免费。

---

### 方案 D：腾讯云 COS + CDN（国内最佳速度）

适合用户主要在中国大陆的情况。

```
GitHub Actions 构建 EXE
  → 通过 coscmd 上传到腾讯云 COS
  → 腾讯云 CDN 加速分发
  → electron-updater generic provider
```

**优势**：
- 中国大陆下载速度最快
- COS 有免费额度（50GB 存储 / 10GB 流量）
- 与 GitHub Actions 集成简单

**费用预估**（少量用户）：
- 存储：87MB × 几个版本 ≈ 免费额度内
- CDN 流量：每月 10GB 免费额度，少量用户够用
- 总计：基本免费或 ~10元/月

**步骤概要**：
1. 注册腾讯云，开通 COS + CDN
2. 创建存储桶，设置公开读
3. 获取 SecretId / SecretKey
4. 在 GitHub Actions 中使用 `coscmd` 上传
5. electron-builder 配置 `generic` provider 指向 CDN 域名

---

### 方案 E：自建服务器 + Nginx（完全自主）

```
GitHub Actions 构建 EXE
  → scp 上传到自有服务器
  → Nginx 提供静态文件服务
  → electron-updater generic provider
```

**优势**：完全自主可控，不限流量

**劣势**：需要维护服务器、配置 HTTPS、处理备份等

## 四、推荐方案及理由

### 🥇 首选：方案 A + 方案 C 组合

```
开发阶段 → 方案 A (GitHub Releases)
    ↓ 用户量大了或有国内用户反馈下载慢
无缝切换 → 方案 C (Cloudflare R2)
```

**理由**：
1. 先用 GitHub Releases 零成本起步，85% 的 Electron 应用都这么做
2. 如果后续需要更好的国内速度，Cloudflare R2 免出口流量费 + 全球 CDN + S3 兼容
3. 两种方案的 electron-updater 配置切换只需要改 `build.publish` 配置

### 🥈 如果用户主要在国内：方案 D（腾讯云 COS）

如果你确定用户 90% 在国内，直接用腾讯云 COS + CDN，速度和稳定性最好。

### 选择决策树

```
你的用户主要在哪里？
  ├─ 海外/混合 → 方案 A (GitHub Releases) 起步，可选升级方案 C (R2)
  ├─ 国内为主 → 方案 D (腾讯云 COS + CDN)
  └─ 有现成服务器 → 方案 E (自建)
```
