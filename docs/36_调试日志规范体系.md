# EasyAgent 调试日志规范体系 v1.0

> **版本**: v1.1 | **日期**: 2026-06-27 | **适用范围**: 全项目（TypeScript / .mjs / .bat / .ps1）

---

## 一、设计原则

| 原则             | 说明                                                      |
| ---------------- | --------------------------------------------------------- |
| **开关可控**     | 所有调试日志必须受开关控制，生产环境默认关闭              |
| **分层清晰**     | 5 级日志（TRACE → DEBUG → INFO → WARN → ERROR），各司其职 |
| **环境变量驱动** | 用 `EASYAGENT_DEBUG` / `LOG_LEVEL` 统一控制，不硬编码     |
| **上下文完备**   | 日志需携带模块名、关键参数，使问题可追溯                  |
| **零开销静默**   | 关闭时 debug 日志不产生任何 I/O 或字符串拼接开销          |

---

## 二、日志级别定义

| 级别    | 值  | 用途                                        | 生产默认 |
| ------- | --- | ------------------------------------------- | -------- |
| `TRACE` | 0   | 极致细节：函数进入/退出、循环迭代、变量快照 | ❌ 关闭  |
| `DEBUG` | 10  | 调试诊断：参数值、中间状态、分支路径        | ❌ 关闭  |
| `INFO`  | 20  | 常规信息：启动/停止、阶段完成、关键操作结果 | ✅ 开启  |
| `WARN`  | 30  | 警告：降级行为、重试、非预期但可恢复的状态  | ✅ 开启  |
| `ERROR` | 40  | 错误：异常、失败操作（必须带错误详情）      | ✅ 开启  |

## 三、统一开关机制

### 3.1 环境变量

| 变量              | 值                                            | 效果                               | 优先级   |
| ----------------- | --------------------------------------------- | ---------------------------------- | -------- |
| `LOG_LEVEL`       | `trace` / `debug` / `info` / `warn` / `error` | 精确设置日志级别                   | **最高** |
| `EASYAGENT_DEBUG` | `1` 或 `true`                                 | 快捷方式，等价于 `LOG_LEVEL=debug` | 次高     |
| （均未设置）      | —                                             | 默认 `INFO` 级别                   | 兜底     |

### 3.2 各语言实现

#### TypeScript（core / server / desktop）

```typescript

const logger = createLogger('MyModule');

// DEBUG 及以下日志仅在 EASYAGENT_DEBUG=1 时输出
logger.debug('详细参数:', { userId, action });

// 如需强制级别（忽略环境变量），显式传入
const verboseLogger = createLogger('Bench', LogLevel.TRACE);
```

**实现位置**: `packages/core/src/utils/logger.ts` → `resolveLogLevel()` 函数

#### .mjs / .cjs 脚本

log.title('Phase 2.5: SQLite3 编译');

## 四、使用方法（快速上手）

### 4.1 开启调试模式

**最简单方式**——设置环境变量后运行任何脚本/命令：

```bash
# ─── Windows CMD ───
set EASYAGENT_DEBUG=1 && build.bat --release
set EASYAGENT_DEBUG=1 && node scripts/unified-sync.mjs

# ─── Windows PowerShell ───
$env:EASYAGENT_DEBUG = "1"
pnpm run start:server
# 或一行
$env:EASYAGENT_DEBUG = "1"; node scripts/rebuild-sqlite3.mjs system

# ─── Linux / macOS ───
EASYAGENT_DEBUG=1 pnpm run start:server
EASYAGENT_DEBUG=1 node scripts/sync-version.mjs
```

**精确控制日志级别**（优先于 `EASYAGENT_DEBUG`）：

```bash
# 输出所有日志（含 TRACE）
LOG_LEVEL=trace pnpm run start:server

# 仅输出 WARN 及以上（屏蔽 INFO/DEBUG/TRACE）
LOG_LEVEL=warn node scripts/unified-sync.mjs

# 仅输出 ERROR 及以上
LOG_LEVEL=error build.bat
```

### 4.2 在各语言中添加调试日志

#### TypeScript（core/server/desktop 代码）

```typescript
import { createLogger, LogLevel } from '@easyagent/core';

// 步骤1: 创建 logger（模块名最好与文件名一致）
const log = createLogger('UploadManager');

// 步骤2: 在关键位置添加日志
export async function uploadFile(filePath: string, url: string): Promise<void> {
  // ✅ 函数入口记录参数
  log.debug('uploadFile 调用', { filePath, url });

  try {
    const stat = fs.statSync(filePath);
    // ✅ 关键变量值
    log.debug('文件信息', { size: stat.size, mtime: stat.mtime });

    const result = await http.put(url, fs.createReadStream(filePath));
    // ✅ 操作结果
    log.info('上传成功', { url, statusCode: result.statusCode });
  } catch (err) {
    // ✅ 错误必须带上下文（三要素：参数 + 错误信息 + 堆栈）
    log.error('上传失败', {
      filePath,
      url,
      error: (err as Error).message,
      stack: (err as Error).stack,
    });
    throw err;
  }
}
```

#### .mjs 脚本

```javascript
import { createLogger } from './lib/logger.mjs';

const log = createLogger('my-build-task');

// 快捷方法速查：
log.title('Phase 1: 编译'); // ═══ Phase 1: 编译 ═══
log.info('正在处理...'); //    ℹ 正在处理...
log.debug('内部变量值:', varName); //    ▶ 内部变量值: xxx  (仅调试模式)
log.ok('编译通过'); //    ✅ 编译通过
log.fail('编译失败'); //    ❌ 编译失败
log.warn('依赖版本过旧，建议升级'); //    ⚠ 依赖版本过旧...
log.error('致命错误', err.message); //    ✗ 致命错误: ...
log.hr(); //    ────────────────────────
```

#### .bat 批处理

```batch
@echo off
chcp 65001 >nul

REM 步骤1: 开头添加开关检测（放在 @echo off 之后）
set _DBG=0
if "%EASYAGENT_DEBUG%"=="1" set _DBG=1

REM 步骤2: 关键信息用 if %_DBG%==1 包裹
if %_DBG%==1 echo [DEBUG] ====== Script Start: %DATE% %TIME% ======
if %_DBG%==1 echo [DEBUG] 当前目录: %CD%
if %_DBG%==1 echo [DEBUG] 目标文件: %OUTPUT_FILE%

REM 步骤3: 正常信息正常输出（不受开关影响）
echo [1/5] 编译 TypeScript...
call npx tsc

REM 步骤4: 错误处理
if %ERRORLEVEL% NEQ 0 (
    echo [ERROR] 编译失败，errorlevel=%ERRORLEVEL%
    if %_DBG%==1 echo [DEBUG] 失败时的工作目录: %CD%
    exit /b 1
)
echo [OK] 编译完成
```

#### PowerShell (.ps1)

```powershell
# 步骤1: 读取开关
$DebugEnabled = $env:EASYAGENT_DEBUG -eq '1'

# 步骤2: 定义调试输出函数
function Write-DebugLog { param([string]$Msg)
    if ($DebugEnabled) { Write-Host "[DEBUG] $Msg" -ForegroundColor Cyan }
}
function Write-Step { param([string]$Msg)
    Write-Host $Msg -ForegroundColor Green
}

# 步骤3: 使用
Write-Step "[1/3] 上传文件..."
Write-DebugLog "源文件: $srcPath → 目标: $destUrl"

try {
    Upload-File -Path $srcPath -Url $destUrl
    Write-Host "[OK] 上传成功" -ForegroundColor Green
} catch {
    Write-Host "[FAIL] 上传失败: $($_.Exception.Message)" -ForegroundColor Red
    Write-DebugLog "完整异常: $($_.Exception)"
    exit 1
}
```

## 五、何时添加调试日志（强制规则）

### 5.1 🔴 必须添加 DEBUG 日志的场景

| #   | 场景              | 日志示例                                                             | 理由                            |
| --- | ----------------- | -------------------------------------------------------------------- | ------------------------------- |
| 1   | **外部命令执行**  | `log.debug('执行命令:', cmd)`                                        | 构建/发布脚本中最常见的排查需求 |
| 2   | **文件 I/O 操作** | `log.debug('写入文件:', filePath, size)`                             | 确定文件是否正确生成            |
| 3   | **错误捕获块**    | `log.error('操作失败', { err: e.message, stack: e.stack, context })` | **必须带上下文**，否则日志无用  |
| 4   | **条件分支**      | `log.debug('进入分支:', branchName, condition)`                      | 确定代码走了哪条路径            |
| 5   | **状态切换**      | `log.debug('状态变更:', oldState, '→', newState)`                    | 追踪状态机流转                  |
| 6   | **API 调用**      | `log.debug('API 请求:', method, url, statusCode, duration)`          | 排查网络问题                    |
| 7   | **关键变量值**    | `log.debug('配置值:', { key1: val1, key2: val2 })`                   | 验证配置是否正确加载            |
| 8   | **构建步骤**      | `log.debug('Phase X 开始, 参数:', params)`                           | 构建脚本的断点                  |

### 5.2 🔴 错误日志必须包含的内容

```typescript
// ❌ 错误：无上下文，无法定位
logger.error('操作失败');

// ✅ 正确：包含完整上下文
logger.error('上传 EXE 失败', {
  file: fileName,
  size: fileSize,
  target: uploadUrl,
  error: e instanceof Error ? e.message : String(e),
  stack: e instanceof Error ? e.stack : undefined,
});
```

### 5.3 不应记录的内容

- ❌ 敏感信息（API Key、密码、token）
- ❌ 超大对象（>1KB 的 JSON）
- ❌ 循环内的无意义计数（除非调试性能问题）
- ❌ 用户输入的个人隐私数据

## 六、代码中日志模式速查

### 6.1 TypeScript 标准模式

```typescript
import { createLogger } from '@easyagent/core';

const log = createLogger('ModuleName');

export async function myFunction(param: string): Promise<Result> {
  log.debug('myFunction 调用', { param });

  try {
    // 关键步骤前
    log.debug('开始处理', { step: 'validate', param });

    const result = await doWork(param);

    // 成功后
    log.info('处理完成', { resultId: result.id });
    return result;
  } catch (err) {
    // 错误必须带上下文
    log.error('myFunction 失败', {
      param,
      error: (err as Error).message,
      stack: (err as Error).stack,
    });
    throw err;
  }
}
```

### 6.2 .mjs 脚本标准模式

```javascript
import { createLogger } from '../lib/logger.mjs';

const log = createLogger('my-script');

function main() {
  log.title('My Script v1.0');

  log.info('步骤 1/3: 检查环境...');
  const config = loadConfig();

  log.debug('配置详情', JSON.stringify(config, null, 2));

  try {
    const result = processData(config);
    log.ok('处理完成');
    log.info('输出文件:', result.outputPath);
  } catch (err) {
    log.fail('处理失败');
    log.error('详细错误', err.message);
    process.exit(1);
  }
}

main();
```

### 6.3 .bat 脚本标准模式

```batch
@echo off
chcp 65001 >nul

REM 检查调试开关
set _DBG=0
if "%EASYAGENT_DEBUG%"=="1" set _DBG=1

if %_DBG%==1 echo [DEBUG] ====== Script Start ======
if %_DBG%==1 echo [DEBUG] CWD: %CD%
if %_DBG%==1 echo [DEBUG] Time: %DATE% %TIME%

echo [1/5] 检查前置条件...
REM 关键步骤用 [DEBUG] 记录参数
if %_DBG%==1 echo [DEBUG] 源文件: %SRC_FILE% 目标: %DST_FILE%

echo [OK] 全部完成
if %_DBG%==1 echo [DEBUG] ====== Script End ======
pause
```

## 六、生产环境保障

### 6.1 默认行为

- **不设置任何环境变量** → 只输出 INFO / WARN / ERROR
- DEBUG / TRACE 级别日志零性能开销（pino 在内部即完成级别过滤）

### 6.2 紧急排障

```bash
# Windows CMD
set EASYAGENT_DEBUG=1
build.bat --release

# Windows PowerShell
$env:EASYAGENT_DEBUG = "1"
.\build.bat --release

# Linux / macOS
EASYAGENT_DEBUG=1 ./build.sh

# 或精确控制
LOG_LEVEL=trace pnpm run start:server
```

### 7.3 发布版本

- `release-publish.bat` 发布流程中不设置 `EASYAGENT_DEBUG`，确保生产静默
- CI/CD 中可在 workflow 配置 `EASYAGENT_DEBUG: '1'` 便于 Actions 日志排查
- Desktop EXE 中，用户通过菜单 "调试 → 启用日志文件" 开关本地文件日志（仅 Desktop 专属）

## 九、迁移指南

### 9.1 现有 .mjs 脚本迁移

将 `console.log / console.error` 替换为 `logger.mjs`：

```javascript
// 迁移前
console.log('开始编译...');
console.error('编译失败:', err.message);

// 迁移后
import { createLogger } from '../lib/logger.mjs';
const log = createLogger('build');

log.info('开始编译...');
log.error('编译失败', err.message);
```

### 8.2 现有 .bat 脚本迁移

在脚本开头添加调试开关检测，在关键步骤添加 DEBUG 输出：

```batch
set _DBG=0
if "%EASYAGENT_DEBUG%"=="1" set _DBG=1

:: 替换裸 echo
:: echo 编译参数: %PARAM%
:: ↓
if %_DBG%==1 echo [DEBUG] 编译参数: %PARAM%
```

### 8.3 迁移优先级

| 优先级 | 文件                           | 理由                     |
| ------ | ------------------------------ | ------------------------ |
| P0     | `build.bat`                    | 最复杂，调试频率最高     |
| P0     | `release-publish.bat`          | 发布关键路径             |
| P1     | `scripts/*.mjs`                | 编译、同步、发布相关脚本 |
| P2     | `packages/server/src/index.ts` | 后端启动/请求日志        |
| P3     | 其他 .ts 文件                  | 按需渐进                 |

## 十、检查清单

开发者在提交代码前需确认：

- [ ] 新增的 try-catch 中，`catch` 块使用了 `log.error` 并携带上下文字段
- [ ] 关键外部调用（exec/HTTP/文件I/O）前后有 `log.debug` 记录参数和结果
- [ ] DEBUG 日志使用 `log.debug`（而非 `log.info`），确保生产环境不输出
- [ ] 无敏感信息泄露（API Key、密码、token 等不记录）
- [ ] .mjs 脚本使用 `scripts/lib/logger.mjs` 而非裸 `console.log`
- [ ] .bat 脚本的 DEBUG 行受 `%_DBG%` 或 `%EASYAGENT_DEBUG%` 开关控制

## 十一、FAQ

**Q: 为什么不直接用 `console.log` 而要用 pino / logger.mjs？**
A: `console.log` 无法按级别过滤。生产环境会输出大量无用信息，且无法通过环境变量统一关闭。

**Q: `EASYAGENT_DEBUG=1` 和 `LOG_LEVEL=debug` 有什么区别？**
A: `EASYAGENT_DEBUG=1` 是快捷方式，等价于 `LOG_LEVEL=debug`。如果同时设置，`LOG_LEVEL` 优先级更高。

**Q: Desktop EXE 中如何开启调试日志？**
A: 菜单栏 → 调试 → 启用日志文件。日志写入 `%APPDATA%/easyagent/logs/easyagent-YYYY-MM-DD.log`。

**Q: 调试日志会影响性能吗？**
A: pino 在日志级别高于当前阈值时会完全跳过（零字符串拼接）。.bat 中的 `if %_DBG%==1` 同理。关闭时零开销。
